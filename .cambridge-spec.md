**与 .cron-spec.md 的关系：** `.cron-spec.md` 已停用（V3.8.3），是国内体系的旧规范。本文件是 V3.9 Cambridge 体系的新规范，独立运作。

## 3. KP 严格匹配纪律

详见 `docs/cambridge_english_kp_draft.md`（共 65 KP，4 类 category）。

**铁律：**
1. 题目 `knowledge_point` 字段必须**严格**等于清单里的 full_path（区分大小写、空格、括号）
2. 不在清单里的 KP 不允许使用，**不要**自行创造新 KP（如要新增，提交 cron-issue.md 而非猜测）
3. KP **全英文**，不出现中文（连解析 explanation 也尽量英文，沉浸式语言环境）
4. **KP 仅 4 类 category**：Vocabulary / Grammar / Reading / Listening。**题型不做 KP 类别**——题型由 `type` 字段（choice/fill）表达，Cambridge Part 由 `chapter` 字段标记
5. 题型归并规则：
   - Word Formation / Key Word Transformations / Open Cloze → Grammar 类 KP
   - MC Cloze (词汇辨析) → Vocabulary 类 KP（`Vocabulary/Word Choice (Cloze)`）
   - Reading MC / Multiple Matching / Gapped Text / Cross-text Matching → Reading 类 KP
   - Vocabulary MC → 直接挂 `Vocabulary/<topic>` KP
   - Grammar MC → 直接挂 `Grammar/<point>` KP

## 4. 题型规范

### 4.1 题型分类（type 字段）

| Question Type | Field | Cambridge Use |
|---|---|---|
| `multipleChoice` (index=0) | choice | Vocab MC, Grammar MC, MC Cloze, Reading MC, Listening, Gapped Text, Multiple Matching |
| `fillBlank` (index=1) | fill | Open Cloze, Word Formation, Key Word Transformations |
| `calculation` (index=2) | calc | （Cambridge 不用，仅数学） |
| `subjective` (index=3) | subjective | **暂不用**（等 AI 评分 API 后做 Writing） |

### 4.1.1 chapter 字段标记 Cambridge Part

题型与 Cambridge 考试 Part 的对应通过 `chapter` 字段表达，不进入 KP：

| Cambridge Part | chapter 字段值 | 主要 KP 类别 |
|---|---|---|
| PET (整体) | `"PET"` | 各类 |
| FCE Part 1 (MC Cloze) | `"FCE Part 1"` | Vocabulary/Word Choice (Cloze) |
| FCE Part 2 (Open Cloze) | `"FCE Part 2"` | Grammar/Open Cloze |
| FCE Part 3 (Word Formation) | `"FCE Part 3"` | Grammar/Word Formation |
| FCE Part 4 (Key Word Transformations) | `"FCE Part 4"` | Grammar/Sentence Transformation |
| FCE Part 5 (Reading MC) | `"FCE Part 5"` | Reading/<technique> |
| FCE Part 6 (Gapped Text) | `"FCE Part 6"` | Reading/Text Coherence |
| FCE Part 7 (Multiple Matching) | `"FCE Part 7"` | Reading/Multiple Matching |
| CAE Part 1-8 | `"CAE Part N"` | 同上对应（Foundation 不出 Writing） |

### 4.2 选择题规范

- 4 选项标准（A/B/C/D）
- options 字段：`["A. xxx", "B. xxx", "C. xxx", "D. xxx"]`（带前缀字母 + 点 + 空格 + 内容）
- answer 字段：单字母 `"B"`（不带 dot/whitespace）
- explanation：1-3 句英文解释为什么是这个答案 + 为什么排除其他选项（精炼）
- **干扰项原则**（Cambridge 标准）：每个错选项必须看似合理但有明确语法/词汇错误。不出"显然错误"的废干扰项。

### 4.3 填空题规范

- content 包含 `____`（4 个下划线）作为答题位置
- answer 字段：用 `|||` 分隔等价写法（如 `"approval|||APPROVAL"`）
- **输入法限制白名单**（feedback_question_quality.md）：
  - 允许的 fill 答案：单词 ≤8 字母 / 简单词组 ≤3 词 / 数字 / 简单标点
  - 不允许：长句 / 整段写作（→ 改 choice 或不出）
- explanation：英文，给出推导逻辑

### 4.4 阅读 Cluster (Reading MC, Long Reading, Gapped Text, Multiple Matching)

- 共享同一阅读材料
- 第一题的 content 字段含完整阅读材料 + 第一题问题
- 后续题的 content 仅含问题（材料引用 group_id 即可）
- 每题独立 group_id（同 cluster 共享）+ group_order (1, 2, 3...)
- 抽题时按 group_id 整组抽 + 按 group_order 顺序展示（V3.8.2 QuestionDao 已支持）

```json
[
  {
    "group_id": "pet_reading_g6_r1_001",
    "group_order": 1,
    "knowledge_point": "Task Types/Reading MC",
    "content": "[passage text 200-300 words]\n\nQuestion 1: ...",
    ...
  },
  {
    "group_id": "pet_reading_g6_r1_001",
    "group_order": 2,
    "knowledge_point": "Task Types/Reading MC",
    "content": "Question 2: ... (refers to passage above)",
    ...
  }
]
```

材料长度：
- PET reading: 150-300 词，3-5 道题
- FCE reading: 300-600 词，4-7 道题
- CAE reading: 500-800 词，6-8 道题

### 4.5 听力题（audio_text）

- 必填 `audio_text`：完整朗读原文（device flutter_tts en-US 朗读）
- audio_text **不要重复 content 里的问题**（音频只朗读对话/独白）
- 角色对话用 `A:` `B:` 前缀
- 单段独白直接写
- content 字段：`"Listen and choose the correct answer.\n\n[问题]"`
- 时长建议：PET 30-60 秒；FCE 60-90 秒；CAE 90-120 秒

### 4.6 词形转换（Word Formation）

- content：句子 + 提示词大写格式 `(APPROVE)`
- answer：变形后的词（注意大小写匹配题目语境）
- 变形规则：noun ↔ verb ↔ adjective ↔ adverb（前缀/后缀加减）

### 4.7 关键词改写（Key Word Transformations）

- content 三行格式：原句 / 关键词 / 模板句（含 `____`）
- 关键词必须用大写显示（FCE 标准）
- 答案 2-5 词（FCE）；2-6 词（CAE）
- **必须用关键词**，不允许避开 key word 转换

```
原句：He started learning piano five years ago.
KEY WORD: FOR
模板句：He ____ for five years.
答案：has been learning piano (5 词，含 FOR)
```

## 5. 难度档（Round）Rubric

### 5.1 Round 1 (Entry)

- Vocabulary: 高频词（前 1500 词，PET R1 限定）
- Grammar: 单一时态、单一规则
- Reading: 显式信息（直接在文中找）
- Listening: 慢速、清晰、单说话人
- Distractors: 显然错误的至少有 2 个

### 5.2 Round 2 (Mid)

- Vocabulary: 中频词（前 3000 词）
- Grammar: 时态变换、复合句
- Reading: 推断信息（结合 1-2 句）
- Listening: 正常语速、对话
- Distractors: 部分合理，需仔细辨别

### 5.3 Round 3 (High)

- Vocabulary: 中高频词（前 5000 词）
- Grammar: 多重语法考点合并
- Reading: 深层推断 + 作者立场
- Listening: 多人对话 + 隐含态度
- Distractors: 全部看似合理，需精确辨析

### 5.4 Round 4 (Bridge / Top)

- Vocabulary: 该 level 上限词（PET 3500 / FCE 5000 / CAE 7000+）
- Grammar: 该 level 最难考点（如 PET R4 = simple cleft / FCE R4 = inversion）
- Reading: 复杂篇章 + 隐喻
- Listening: 含口音 / 快速 / 复杂主题
- Distractors: 仅一个微小区别，类似 Cambridge 真题最难题

## 6. 答案兼容性（alt_answers）

继承 V3.7.1 alt_answers 机制：

```json
{
  "answer": "approval|||APPROVAL"
}
```

**Cambridge 必备 alt_answers 场景：**
- Word Formation 大小写：`"approval|||APPROVAL"`
- 美式/英式拼写：`"colour|||color"`、`"organize|||organise"`
- 缩写/全写：`"don't|||do not"`、`"won't|||will not"`
- 标点变体：`"the police are|||the police're|||the police is"`（如多种语法接受）

---

## 7. 输入法限制白名单

继承 feedback_question_quality.md 的 V3.7.5 规则。Cambridge 体系下：

**允许 fill 的：**
- 英文单词 ≤8 字母（rabbit, hospital）
- 短词组 ≤3 词（give up, take off）
- 词形转换答案（approval, disagreement）
- 数字 / 时间 / 日期（12, 3 PM, Monday）
- 关键词改写（5-6 词的转换句）

**不允许 fill 必须改 choice：**
- 完整句子答题（除 Key Word Transformations 外）
- 解释/解析类答题
- 任何含特殊符号（#&%@等）

## 9. PET R1 分布参考

均匀分布，**20 KP × 10 题 = 200 题**。
- Vocabulary 6 × 10 = 60
- Grammar 9 × 10 = 90
- Reading 3 × 10 = 30
- Listening 2 × 10 = 20

详细分配见 `docs/cambridge_english_kp_draft.md` 第 7 节。

**Stage 1 (PET R1) 出题流程示例：**
1. 读本文件 + cambridge_english_kp_draft.md
2. 列出 24 个 KP + 每个分配的题数
3. 按 KP 顺序生成（先词汇 6 KP × 8 题 = 48 题；再语法 9 × 8 = 72 题；后续同理）
4. 输出到 `batch_2026_05_XX_g6_english_pet_r1.json`
5. 双写到 `assets/data/batches/` 和 `question_bank/`
6. 注册到 `question_bank/index.json`（version 5+）和 `lib/main.dart` 的 `_bundledBatchAssets`
7. flutter analyze（无 error）+ git add + git commit + git push

## 11. Agent 并行约定

- 每 stage（200 题）用 1-2 Agent 并行
- 单 Agent 单 stage（不要让 1 个 Agent 跨 stage）
- Agent prompt 模板：
  ```
  按 .cambridge-spec.md 出 [Stage X: PET/FCE/CAE level] 的 200 题。
  KP 分布：[列表]。
  输出 JSON 文件 batch_YYYY_MM_DD_gN_english_[level]_rN.json。
  严格遵守输入法限制 + KP 全英文匹配 + alt_answers 等价写法。
  生成完毕后抽 5 题自我试做检查质量。
  ```
- Token 预估：200 题 ≈ 100K-150K tokens / Agent
