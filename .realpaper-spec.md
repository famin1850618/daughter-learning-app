## 1.4 Issues & Resolutions 反馈机制（V3.12.7 强制）

**每轮入库 agent 完成后必须附 "Issues & Resolutions" 报告**，主 session 整理交 Famin 决策是否写入纪律。

**目的：** 从一线 agent 的纠结点识别新规则，让 spec/skill 体系不断细化（observation loop O5/O6 实施）。

**报告格式：**

```
## Issues & Resolutions

### 问题 N: <简述>
- **遇到场景**: 第 X 卷处理时 / 第 Y 题判定时
- **现象**: 具体描述（含原题片段 + agent 当时尝试）
- **当时决策**: 我选了 X，依据 spec §A.B / 自由判断
- **是否合规**: ✅符合 / ⚠ 边界 / ❌违反但权宜
- **建议是否写入纪律**: 是/否 + 理由
```

**报告要求：**
- 全卷处理中**每一个判断犹豫的点**都列出
- 包括但不限于：题型识别歧义 / KP 不在清单边界 / 形式 A vs B 边界 / 图截图选择 / round 判定犹豫 / 加点字识别失败 / 拼音是否带声调判断
- "全顺利没问题"= 思考不深入的**失败信号**
- 所有"我判断了但不确定对不对"的点全暴露

**主 session 处理流程：**
1. 收 agent Issues 清单
2. 整理去重 + 按类型分组
3. 交 Famin 决策（每条标"建议写入纪律 / 个案处理 / 不需要"）
4. Famin 同意写入的 → 回填对应 spec 节 + skill rules/

**触发条件：** 每轮 agent 完成强制执行，不允许跳过。

## 1.5.1 阅读理解原文完整性（V3.12.12 硬约束，全科适用）

> **组合题依托于同一篇原文的，每一题都完全显示原文，只换题目。**

### 严禁的"画蛇添足精简"

| ❌ 反例 | 正例 |
|---|---|
| `阅读《向日葵》全文（约 1100 字散文，叙述「我」与女儿在生日前后的情感变化：①早上...）` | `阅读下面文章，回答问题。\n\n[完整 1100 字原文]\n\n问：...` |
| `阅读《我爱家乡的秋夜》（散文，约 600 字。开篇：「家乡的秋夜素雅、幽静...」首先描写月光下水墨画...）` | `[完整 600 字原文]\n\n问：...` |
| `阅读《XX》(节选，详见前文)` | `[完整原文]\n\n问：...` |
| 同一篇文章 N 道题，content 各自精简描述 | 同一 group_id 内 N 道题 content **完全相同**（原文部分），仅 question 不同 |

### 强制做法（agent 入库时）

1. **完整粘贴原文** —— 从 raw.txt / PDF 渲染图复制全部，不删段落、不写"省略"
2. **同 group_id 内 content 共享原文段** —— 第 1-N 题 content 第一部分完全一致
3. **`问：` 之后才是题目** —— 用 `\n\n问：...` 分隔原文与题目，便于解析
4. **不允许"内容简介"** —— 任何 `约 N 字`、`叙述...`、`首先...接着...` 的精介绍都禁止
5. **不允许"详见前文"** —— 每道题 content 必须自包含

### 适用范围

- **语文阅读理解**：散文、说明文、议论文、记叙文等
- **英语阅读**：长篇 reading comprehension cluster
- **数学/物理/化学应用题组合**：同 group_id 题情境部分 content 应一致
- **历史/地理**（未来扩展）：材料题

### 反例（V3.12.10 实测发现，V3.12.12 全部修复）

```
  → V3.12.12 修：恢复完整原文

qimo_quanzhen_001 group qimo_quanzhen_001_q16 (3 道) 《记忆里的香甜》
  → V3.12.7 入库写"约 800 字。回忆「我」幼时..." 简介
```

## 1.5 原题保真原则（V3.12.7 顶级硬约束，违反整题撤回）

> **入库 = 原题 1:1 转 batch JSON。除 OCR 矫正错字 + LaTeX 公式化外，禁止任何对原题内容的加工。**

### 严禁的 Agent 加工行为

| 加工行为 | 例子 | 危害 |
|---|---|---|
| ❌ **加释义/翻译/解释** | `ji mo（孤单冷清）` 加括号释义 | **直接剧透答案**（违反 quality_rules §1） |
| ❌ **去除原题标记** | 拼音 `jī mò` → `ji mo`（去声调） | 学生本来要看声调辨析，去掉降低难度 |
| ❌ **添加格式提示** | `（不带声调）` `（填两个词）`（如原题没说） | 改变原题难度 |
| ❌ **改写题面措辞** | `读拼音写词语` → `短文中拼音写字词` | 不是原题文字 |
| ❌ **拆同组合题为独立 question 不用 group_id** | 一段短文 N 个空切 N 道独立题 | 破坏原题语境，UI 不能整组连续做（违反 §5.4） |
| ❌ **题面包含答案** | 拼音后括号写词、"图中三角形（直角三角形）" | 剧透（违反 quality_rules §1 模式 1/3） |
| ❌ **简化原题语料** | 阅读理解原文截断 | 改变题难度（违反 §9.1 完整性优先） |

### 允许的修改（仅这 5 种）

1. **OCR 错字矫正**：明显的 OCR 错字（如"杀"误识为"杂"）按上下文矫正
2. **LaTeX 公式化**：纯文本数学表达 → `$\frac{1}{3}\pi r^{2}$`（§5.7）
3. **加点字 markdown 加粗**：`粽` (加点) → `**粽**`（§5.6）
4. **JSON 字符串转义**：双引号 `"..."` → `「...」`（§5.5）
5. **图嵌入**：`如图` 题加 `image_data` 截图（§9.2）

**除此之外都不允许**。原题怎么写就怎么入库。

### 4a annotate 阶段自检（pre_commit_check.md 第 13 项）

```python
def check_no_agent_doctoring(content: str, original_raw: str) -> list:
```

## 2. 9 步 Pipeline（V3.12.5 起 round 判定与 annotate 分离）

```
真题文件（.doc/.docx/.pdf）
  └→ [1 extract]    libreoffice headless --convert-to txt → .cache/realpaper/<sha1>/raw.txt
  └→ [2 segment]    正则切题（"^一、""1.""(1)"等模式 + 大题套小题用 group_id）→ stems[]
  └→ [3 match-ans]  找"参考答案/答案"段；或独立答案文件配对
  └→ [4a annotate]  Claude 会话级标注：type / chapter (∈158) / KP (∈491) / answer / explanation
                    ⚠ round 字段强制留 null，不许 LLM 自由判
  └→ [4b round]     按科目当前阶段判 round（见 §2.1 三阶段表）：
                      - 阶段 0（无锚点）：LLM 自由推理判
                      - 阶段 1（有锚点+rubric）：LLM 必须读 anchors_*.json + rubric_*.md 判
                      - 阶段 2（skill/脚本成熟）：强制脚本（如数学 d2_math_review_v4.py）
  └→ [5 validate]   py 脚本校验 chapter/KP/题型一致性 + KP 不在清单的题入等候区
  └→ [6 emit]       batch JSON 双写 assets/data/batches/ + question_bank/
  └→ [7 register]   question_bank/index.json + main.dart _bundledBatchAssets
  └→ [8 commit]     git commit + push + 更新 manifest
```

每步详细见 §10 实施细则。

### 2.1 Round 判定三阶段（V3.12.5 起硬规则）

每科 round 判定按"渐进沉淀"演化，**绝不退化**（每次 Famin verify 反馈后必须立即沉淀，让下批扫描受益）。

#### 三阶段定义

| 阶段 | 触发条件 | round 判定方式 | Famin 角色 |
|---|---|---|---|
| **0 无锚点** | 该科目首次扫描 / 锚点尚未建立 | LLM 自由推理判 | 抽审 verify markdown（锚点候选 + 改档）|
| **1 锚点+rubric** | 首轮 verify 反馈完成 → 锚点 + rubric 文档已沉淀 | **LLM 必须读 `anchors_*.json` + `rubric_*.md` 判**（不许凭空判，必须引用锚点 ID）| 继续抽审，反馈累积 |
| **2 skill/脚本** | rubric 稳定 + 累计 ≥ 3-4 轮 verify + 75% 严格对齐 | 脚本强制（python tools/d2_*_review_v*.py）| 仅 flag_review（high_variance / anchor_disagree）抽审 |

#### 各科目当前阶段（2026-05-08）

| 科目 | 阶段 | 判定方式 | 文档资产 |
|---|---|---|---|
| **数学** | ✅ 阶段 2 | `tools/d2_math_review_v4.py` 强制 | `~/.claude/skills/difficulty-math/` (anchors+evaluate+rubric+scripts) |
| **语文** | ⏳ 阶段 0→1（首轮 verify 进行中）| 等 Famin 审完 → 升阶段 1 | `docs/anchor_questions_g6_chinese.json`（20 道未审）+ rubric 待 Famin 审完后沉淀 |
| **英语** | ✅ 阶段 2（特殊路径）| Cambridge composition skill 自带 round（A2/B1/B2/C1 → R1/R2/R3/R4）| `~/.claude/skills/composition-english/`（待建）|
| **物理** | ⏳ 阶段 0 | LLM 自由判 | 待首轮扫描后建立锚点 |
| **化学** | ⏳ 阶段 0 | LLM 自由判 | 待首轮扫描后建立锚点 |

#### 阶段升级触发流程（强制）

每次 Famin verify 反馈后**立即**执行：

```
verify markdown 收到反馈
  ↓
1. 锚点更新：把同意的锚点写入 docs/anchor_questions_<grade>_<subject>.json
              + _famin_review.comment 字段
  ↓
2. Rubric 沉淀：把 reasoning 抽提到 docs/rubric_<subject>.md（首次→新建）
              重点记录"什么档对应什么特征" + "Famin 推翻 LLM 的典型 case"
  ↓
3. 反向修订：revert 项 → 改 batch round 字段（双写）+ commit
  ↓
4. 阶段判定：
     - 首轮 verify 完成 → 阶段 0 → 阶段 1（下批扫描 prompt 必须引用锚点+rubric）
     - 累计 ≥ 3-4 轮 verify + 75% 严格对齐 → 阶段 1 → 阶段 2（生成 reviewer 脚本 + skill）
  ↓
5. 工作流文件同步：
     - .realpaper-spec.md §2.1 各科阶段表更新
     - feedback_round_judge.md 同步
     - 新建 skill 时把锚点+rubric+脚本打包到 ~/.claude/skills/difficulty-<subject>/
```

**审了不沉淀 = 白审**。这是 feedback_strategy_to_execution 的硬要求。

#### 禁止行为

- ❌ 入库 agent 在 4a 阶段直接给 round 字段赋值（哪怕"为方便 4b 跑"）
- ❌ 数学科 LLM 自由判 round（已阶段 2，必须跑脚本）
- ❌ 已进入阶段 1 的科目，agent prompt 不引用锚点+rubric 凭空 LLM 判（这是退化）
- ❌ 跳过 verify 直接把 LLM 判的 round 落库（V3.12.5 commit 161072d 反面教材）
- ❌ Famin verify 完不沉淀就启动下批扫描（必须先沉淀再扫）

## 5. 题型识别规则

| 模式 | 判定 type |
|---|---|
| 题面含 4 行 `A./B./C./D.` | choice |
| 题面是单一陈述句 + `(    )` 或 `判断对错`（"对的画√错的画×"等指引）| **judgment**（V3.10 新）|
| 题面含 `_____` 或 `（  ）`且无 ABCD 选项；答案 ≤ 8 字 | fill |
| 题面含 `计算下列` `解方程` `列式` `求...的值` | calculation |
| 题面含 `作文` `根据材料写` `谈谈你的看法` `写一篇` `不少于 X 字` `谈谈感受` | **subjective**（保留区，source 加 `_subj_held`）|

### 5.1 输入法限制白名单（feedback_question_quality.md 继承）
fill 答案必须满足：
- 中文：单字 / ≤4 字短词 / ≤5 字古诗原文 / 修辞名词 / 作家作品名
- 数字：纯数字 / 简单分数（"1/3"）/ 简单比例（"3:1"）
- 不允许：长句 / 整段表达 / 含特殊符号（π/²/×/拼音声调等 → 改 choice）

### 5.2 主观题归类（V3.12.22 M2 强化）

**统一处理原则**：任何**长答案 / 开放题 / 语义评分**都用 `type=subjective` + source `_subj_held` 后缀。

**触发判断**（agent 入库时必检）：

| 信号 | 例子 | 处理 |
|---|---|---|
| 答案 > 4 字（中文）/ > 8 字母（英文）非短词 | "沸水敢蹚，烈火敢踏。比喻不避艰险..." | → subjective |
| 答案是整段描述/分析 | "请简述实验步骤" / "请谈谈你的看法" | → subjective |
| 答案需语义判分（同义不同字面）| "请用自己的话概括..." | → subjective |
| 含"简述/谈谈/请用自己的话/写出..." | 语境提示开放表达 | → subjective |
| 作文/写作 | "根据材料写一篇..." / "不少于 400 字" | → subjective |

**禁止**：
- ❌ 长答案当 fill 入库（学生输入法限制 + AnswerMatcher 字面匹配过严，体验差）
- ❌ 跳过主观题不入库（保留题源很重要，等 AI 评分 API 接入后启用）

**Source 后缀**：
- 常规 source: `realpaper_g6_chinese_bubian_qm_nanshan_001`
- 主观题区: `realpaper_g6_chinese_bubian_qm_nanshan_001_subj_held`

agent 入库时主观题题写到带 `_subj_held` 后缀的独立 batch JSON（避免污染主抽题池）。

**V3.12.21 实测**：xsc_baoan #8(3)(4) 文言文成语推断长答案被错入 fill，应转 subjective。

旧的范围规则保留：
- 政治/历史的简答（不在产品范围）→ 跳过整题
- 数学/物理/化学的简答（如"简述实验步骤"）→ subjective + `_subj_held`

### 5.3 判断题（V3.10 新）
- type=`judgment`；answer 必须是 `"对"` 或 `"错"`（单字）
- AnswerMatcher 接受等价写法：对/正确/√/T/true/yes 视为"对"；错/错误/×/F/false/no 视为"错"
- options 字段填 `null`（UI 自动渲染对/错二选一按钮）
- alt_answers 不需要（matcher 已内置归一化）

### 5.4 系列组合题纪律（V3.12.5 全局原则，适用所有科目）

**总原则（绝对禁令）：**

> **绝不把一道大题拆成多道完全独立的题。** 必须保留共同题面，用 `group_id` 关联形成系列组合题。

适用所有科目（数学/语文/英语/物理/化学），以及所有有"共同上文"的复合结构：
- 大题指令 + 多个并列子问题（如"在下列句子中填关联词：(1)...(2)...(3)..."）
- 候选池 + 多个语境填空（如"从下列成语中选填：成语 A/B/C... (1)...(2)..."）
- 阅读材料 + 多个理解小题（如"阅读下文后回答：(1)...(2)...(3)..."）
- 图/表/材料 + 多个分析小题
- 数学应用题"已知条件 + 多问"（如"(1) 求...; (2) 在 (1) 的基础上...; (3) 拓展..."）
- 英语 cloze test、reading comprehension 多 question
- 物理化学的实验题、综合题

**两种处理形式（按子题"形式统一性 + 长度"判断）：**

### 形式 A：压成 1 道多空 fill（V3.12.7 优先选）

**适用条件（必须全部满足）：**
- 所有子题**同一题型**（全 fill 或全 choice）
- 所有子题**同一题面指令**（如全部"看拼音写词语"，全部"填关联词"，全部"古诗背诵默写"）
- 子题文本短，答案是离散单字/短词（≤4 字）
- 答案之间无依赖关系（不是"题(1) 求 X，题(2) 用 X 求 Y"）

**入库格式：**
```yaml
type: fill
content: "[共同题面 + 列出所有空]"
answer: "答案1,答案2,答案3,..."  # 用逗号分隔，按原题顺序
```

**举例：**
```yaml
原题: "读拼音写词语。chú xī(___) chūn lián(___) rán fàng(___) biān pào(___)..."

入库 (1 道 fill):
  type: fill
  content: "读拼音写词语。chú xī(   ) chūn lián(   ) rán fàng(   ) biān pào(   )..."
  answer: "除夕,春联,燃放,鞭炮,..."
```

UI 学生输入"除夕,春联,燃放,..."一次答完整题。AnswerMatcher 已支持逗号分隔判错（V3.7 起）。

### 形式 B：N 道独立 question + group_id（V3.12.5 沿用）

**适用条件（任一满足即用）：**
- 子题**题型混合**（fill + choice 混合，如"先补全词语再选词填空"）
- 子题**长度长**（如阅读理解每子题独立段落 + 长答案）
- 子题**答案有依赖**（题(2) 基于题(1) 答案）
- 含**复杂候选池**（候选池 ≥ 子题数，需独立 choice）
- 含**阅读理解 / 长材料**（每子题保留完整原文）

**统一字段：**

| 字段 | 含义 |
|---|---|
| `group_id` | 同一大题所有子题共享（如 `"qizhong_002_q15"`）|
| `group_order` | 子题顺序（1, 2, 3, ...）|
| `content` | **必须含共同题面**（重复也保留），后接当前子题文字 |
| `type` | 子题类型独立判（fill / choice / judgment 可混合）|

**🔴 V3.12.21 新硬纪律：小题序号统一格式**

每个子题 content 末尾的"当前小题"行必须用 **`N. 题面`** 格式（半角数字 + 半角句号 + 半角空格 + 题面）。**所有学科统一**。

| 原题格式 | 入库统一格式 |
|---|---|
| `(1) 过新年...` | `1. 过新年...` |
| `（一）作家是谁` | `1. 作家是谁` |
| `① 第一个...` | `1. 第一个...` |
| `1．（5分）...` | `1. (5 分) ...` |
| `1) 第一题` | `1. 第一题` |
| `A. 候选成语` | `1. 候选成语`（如果是子题）|

**Why**：
- 子题序号要跟 `group_order` 字段值一致（group_order=1 → "1." / group_order=2 → "2."）
- 学生 UI 上看到清晰序号，不被原题各种 (1)(一)①A. 格式干扰
- 错题集按大题聚合时序号一致便于复盘
- agent 入库时**强制规范化**（属于 §1.5 原题保真允许的"格式标准化"修改，跟"OCR 矫正错字"同级）

**§1.5 原题保真豁免补充**：小题序号格式归一化是**允许的修改**（与"加点字 markdown 加粗""LaTeX 公式化"同级），不算违反原题保真。原始文字（题面内容）必须 1:1 保留。

举例（阅读理解保留完整原文）：
```yaml
所有子题 group_id="qizhong_002_q15"，type=choice：

子题 1 (group_order=1):
  content: "阅读《人类能在地球上生活多久》：[完整 700 字说明文]\n问：'来自人类自身'的威胁主要指什么？"
  options: [...]
  answer: "D"

子题 2 (group_order=2):
  content: "阅读《人类能在地球上生活多久》：[完整 700 字说明文]\n问：'这一过程'指什么？"
  options: [...]
  answer: "B"
```

### 选 A 还是选 B 的判断流程

```
所有子题同一题型且题型为 fill?
├─ 否 → 选 B
└─ 是 → 所有子题同一题面指令?
        ├─ 否 → 选 B
        └─ 是 → 所有答案 ≤4 字 + 无依赖?
                ├─ 否 → 选 B
                └─ 是 → 选 A（压 1 题多空 fill）
```

### 5.4.1 拆题硬子规则（V3.12.13 全科强制，§5.4 的可执行细化）

§5.4 总原则的 4 条硬子规则，4a annotate 阶段强制执行。每条违反 = 整 group 撤回。

#### 子规则 1：同 group_id 题在 batch JSON 数组中**索引连续**

```python
# 错误（V3.12.7 实测发现，数学拆题违规）

  {group_id: "g1", group_order: 1, ...},  # 大题1.子题1
  {group_id: null, ...},                   # ❌ 中间插了独立题

# 正确

  {group_id: "g1", group_order: 1, ...},
  {group_id: "g1", group_order: 2, ...},  # 紧跟
  {group_id: "g1", group_order: 3, ...},
  {group_id: null, ...},                   # 独立题在 group 之后
```

**自检：**
```python
def check_group_continuous(batch):
    cur_gid = None
    seen_gids = set()
    for q in batch.questions:
        gid = q.get('group_id')
        if gid != cur_gid:
            if gid is not None and gid in seen_gids:
                return f'❌ group_id={gid} 不连续（中间被打断）'
            if cur_gid is not None:
                seen_gids.add(cur_gid)
            cur_gid = gid
    return None  # OK
```

#### 子规则 2：同 group 子题 content **第一部分（情境段）完全相同**

同 §1.5.1 阅读理解：同 group 子题情境部分必须 byte-identical，仅 `\n\n问：...` 之后的 question 部分不同。

**自检：**
```python
def check_group_prefix_consistent(batch):
    groups = group_by_group_id(batch.questions)
    for gid, items in groups.items():
        if len(items) < 2: continue
        # 取每道 content 的"问：" 之前部分
        prefixes = [c.split('问：')[0].rstrip() for c in [q.content for q in items]]
        if len(set(prefixes)) > 1:
            return f'❌ group_id={gid} 情境段不一致'
    return None
```

#### 子规则 3：后续子题 content 不能含前面子题的 answer

V3.12.4 §1 模式 4 的同源问题：剧透关键中间步骤会降低难度。

例（违规）：
```yaml

子题 1: content="(1) 求六年级学生人数", answer="120"
子题 2: content="(2) 已知六年级 120 人...求总人数"  # ❌ 题面把 (1) 答案 120 写出来了
```

修法：
```yaml
子题 2: content="(2) 在 (1) 求得的六年级人数基础上...求总人数"  # ✓ 引用关系不剧透
```

**自检：**
```python
def check_no_prev_answer_leak(batch):
    groups = group_by_group_id(batch.questions)
    for gid, items in groups.items():
        items.sort(key=lambda q: q.group_order)
        for i, q in enumerate(items[1:], 1):
            for prev in items[:i]:
                for ans_seg in prev.answer.split('|||'):
                    if ans_seg.strip() and ans_seg.strip() in q.content:
                        return f'❌ group_id={gid} 子题 {q.group_order} 含子题 {prev.group_order} 答案'
    return None
```

#### 子规则 4：拆题阈值（form A vs form B）

| 条件 | 阈值 | 选 form |

| 子题数 | > 6 | B 连续多题 |
| 答案长度 | 平均 ≤ 4 字 | A |
| 答案长度 | 平均 > 4 字 | B |
| 题型混合 | fill + choice / fill + calc | B |
| 子题间答案依赖 | 有依赖 | B |
| 含图/长材料 | 有 | B |

**默认偏好 form A**（V3.12.7 起）：能压成 1 道多空 fill 就压。形式 B 仅当上述任一条件触发。

### 5.7 LaTeX 公式化纪律（V3.12.7 全科强制，主要影响数学/物理/化学）

**总原则：** 数学题里**所有数学表达必须用 `$...$` LaTeX 公式化**，禁止纯文本写分数/比例/上下标/特殊符号。

**强制公式化的写法：**

| 纯文本（错误）| LaTeX（正确）| 渲染 |
|---|---|---|
| `1/3` | `$\frac{1}{3}$` | 分数 |
| `1/3πr²h` | `$\frac{1}{3}\pi r^{2} h$` | 体积公式 |
| `2:3` 比例 | `$2:3$` 或保留 `2:3` 在 `$...$` 内 | 比例 |
| `r²` | `$r^{2}$` | 上标 |
| `²` `³` `½` | `$^{2}$` / `$^{3}$` / `$\frac{1}{2}$` | 各种符号 |
| `π` 单字 | `$\pi$` 或 content 含 π 字符（V3.7.9 渲染保留 π）| 圆周率 |
| `×` `÷` | `$\times$` / `$\div$`（在公式内）| 乘除 |
| `≠` `≤` `≥` | `$\neq$` / `$\leq$` / `$\geq$` | 不等号 |

**MathText widget（flutter_math_fork）仅识别 `$...$` 单美元符**，禁止：

- ❌ `$$...$$`（双美元符是 LaTeX 块级公式语法，不支持）
- ❌ `\(...\)` 或 `\[...\]`（不支持）
- ❌ 纯文本含分数线/上标/比例（V3.12.6 数学 5 处明文分数 + 1 处比例反面教材）
- ❌ `\colon`（flutter_math_fork 不支持的扩展命令，用 `:` 替代）—— V3.12.8 实测 153 处违反

**单位上标允许保留 Unicode 字符**（cm²/m³/dm² 等）：
- ✅ `cm²` `m³` `dm²` `kg/m³` 等单位组合直接 Unicode 字符（Flutter Text 正常渲染）
- ❌ 变量上标必须公式化：`r²` → `$r^{2}$`，`x² + y²` → `$x^{2} + y^{2}$`
- 区别：单位（cm/m/kg/L/t）+ ²³ 用 Unicode；变量（r/h/x/y/a/b）+ ²³ 用 LaTeX

**4a annotate 阶段必检：**

```python
import re
def has_unformatted_math(text: str) -> list:
    """扫 content/explanation 找未公式化的数学表达"""
    issues = []
    # 把 $...$ 内的内容剔除后检查
    stripped = re.sub(r'\$[^$]+\$', '', text)
    # 明文分数 a/b（数字间）
    if re.search(r'(?<!\d)\d+/\d+(?!\d)', stripped):
        issues.append('明文分数')
    # 明文上标 ²³
    if re.search(r'[²³⁴⁵]', stripped):
        issues.append('明文上下标')
    # 含 r² h³ 等不在公式内的字母+上标
    return issues
```

#### 5.6.6 句子级强调（波浪线 / 着重号 / 画线 / 加黑，V3.12.14 全科适用）

4. **UI 渲染**：MathText 解析 `**...**` 句子级也加粗，与字符级共用

```python
def check_wave_line_emphasis(question):

        # 找句子级 markdown（5+ 字 ≠ §5.6 字符级 1-2 字）

    return None
```

#### 5.6.7 题面措辞与渲染同步（V3.12.17 新增硬约束）

题面里的指代措辞**必须**跟 app 实际渲染方式一致。原题说"加点字"/"画波浪线"，但 app 用 `**X**` markdown 渲染成**加粗**——题面措辞要同步改成"加粗"。

**强制替换表（content + explanation 双扫）：**

| 原题措辞 | app 渲染 | 题面应改为 |
|---|---|---|
| 加点字 / 加点的字 / 加点词 | markdown 加粗 | 加粗字 / 加粗的字 / 加粗词 |
| 画波浪线的句子 / 波浪线的句子 | markdown 加粗 | 加粗的句子 |
| 画线句子 / 划线句子 / 画线的句子 | markdown 加粗 | 加粗的句子 |
| 着重号 | markdown 加粗 | 加粗（句子或字）|

**Why:** 用户在 app 里看到加粗，没看到点/线/号，题面说"加点字"会让用户困惑。

**操作（任何渲染改造完成后**当天**扫全 batch）：**
```python
patterns = [
    (r'加点字', '加粗字'),
    (r'加点的字', '加粗的字'),
    (r'加点词', '加粗词'),
    (r'加点的「', '加粗的「'),
    (r'画线句子|划线句子|画波浪线的句子|波浪线的句子|画线的句子|划线的句子', '加粗的句子'),
]
```

**explanation 中保留辅助说明**："（**加粗字**为原题加点字）"——这种自我说明短语保留，让用户理解"原题是加点，app 用加粗替代"。

**反例链：**
- V3.12.7 加点字改 markdown 加粗 → 题面没改 → V3.12.16 才扫一波
- V3.12.16 扫了"波浪线"漏了"画线" → V3.12.17c 补一波
**纪律：** 渲染改造**当天**扫全 batch + 周期性回头扫漏的措辞。详见记忆 `feedback_data_consistency.md`。

### 5.5 复合题：fill + choice 类型混合（§5.4 的子情况）

**适用结构：** 大题先给一组待补全的词/句（A 部分），再用补全后的词去填语境空（B 部分）。典型如部编六下 d1 单元卷 1 第 4 大题：

```
4. 把下列四字词语补充完整，并选词填空。
   万(  )(  )新   (  )(  )不同   万不(  )(  )
   (  )灯(  )彩  (  )(  )分明   两(  )三(  )
   (1) 春节到了，中华大地上到处____，显出一派____的气象。
   (2) 在外边做事的人，除非____，必定赶回家来吃团圆饭。
   (3) 少数民族和汉族的饮食风俗虽然____，但各族人民热爱祖国、热爱民族大团结的心是相同的。
```

**遵循 §5.4 全局原则：所有子题用同一个 `group_id`（不分 _a / _b）**，type 可不同：
- A 部分子题 type=`fill`（补全字形）
- B 部分子题 type=`choice`（候选池 = A 部分补全后的词）

```yaml
group_id: "d1_kp1_q4"  # 整大题共享

A 部分（补全题，group_order=1..6）:
  type: fill
  content: "下列大题第 4 题为补全词语并选词填空。先补全 6 个四字词语，再用补全后的词回答 (1)(2)(3) 三句。当前小题：补全四字词语「万(  )(  )新」"
  answer: "象更"

B 部分（选词填空，group_order=7..9）:
  type: choice
  content: "下列大题第 4 题为补全词语并选词填空。补全后的 6 个成语：万象更新 / 截然不同 / 万不得已 / 张灯结彩 / 黑白分明 / 两面三刀。当前小题：(1) 春节到了，中华大地上到处(    )，显出一派___的气象。本题选第 1 个空。"
  options: ["万象更新", "截然不同", "万不得已", "张灯结彩", "黑白分明", "两面三刀"]
  answer: "张灯结彩"
```

**关键纪律：**
1. **同一 group_id**（不要分 _a / _b 子组，§5.4 已明确）
2. **每个子题 content 必须包含共同题面**（"下列大题第 4 题为补全词语并选词填空..."）
3. A 部分考字面识记 → fill
4. B 部分考候选辨析 → choice + options 显式列出 A 部分答案集

**多语境空的处理：**
- 一句含 N 个空（如"到处___，显出一派___"）→ 拆 N 道 choice，content 标"本题选第 K 个空"
- 候选池可重复使用 → options 列全部 6 个；候选池一次性消耗 → content 注明"每个成语只用一次"

**小变体：**
- A 部分极少（≤2 道）→ 仍拆 type，不要合并
- A 部分 = 课内默写（如"补全古诗：少壮不努力，___"）+ B 部分语境填 → A 仍 fill；B 候选池 = A 答案集

**判定 round：**
- A 部分 fill：字面识记，多数 R1
- B 部分 choice：候选池辨析，一般 R2；候选数 ≤ 3 且语境直白可降 R1

**实施位置：** Pipeline §2 第 2 步 `segment.py` 识别"...补充完整，并选词填空"或"...选择恰当的成语填入下列各句"等大题指令，按本节拆。无法判定时整道入"等候区"等人工审核。

## 6. 难度档（round）判定

由 Claude 主观判，基于 3 个维度：

### 6.1 试卷类型基线

| 试卷类型 | 整卷 round 基线 |
|---|---|
| 单元/小测 | R1 |
| 月考 | R1-R2 |
| 期中/期末 | R2-R3 |
| 中考真题 | R2-R4 全 |
| 培优/竞赛 | R3-R4 |

### 6.2 卷内位置加权
- 卷首选择前 1/3：基线 -1（最低 R1）
- 卷面中段：基线
- 卷尾大题/压轴：基线 +1（最高 R4）

### 6.3 KP 复杂度
- 单 KP 题：保持上述判定
- 跨年级综合题：**取最高年级 KP**（强制规则）
  - 7 + 9 年级 KP → 归 9 年级 KP
  - 7 + 8 + 9 → 归 9 年级 KP

### 6.4 round 含义（V3.8 沿用）
- R1：基础（教材内单一 KP 直接套用）
- R2：中等（同章节多 KP 综合）
- R3：较难（跨章节综合 / 应用题情境）
- R4：竞赛/压轴（跨年级综合 / 创新探究 / 拓展深度）

### 6.5 组合题难度统一（V3.12.5 全科强制）

**规则：** 同 `group_id` 的所有子题，最终 round = `ceil(各子题原始 round 平均值)`，统一写回每个子题的 `round` 字段。

**适用：** 所有科目（数学/语文/英语/物理/化学），所有 round 判定阶段（脚本判 / LLM 判 / 自由判）。

**为什么：**
- 系列组合题（§5.4）UI 上按 `group_id` 整组连续呈现给学生，整组应有"统一难度感受"
- 子题难度差异大时（如 R1+R2+R4），整组体验受最难子题主导，向上取整体现这点
- 错题集按"大题"聚合复盘，整组用同一个 round 标签更直观
- 抽题策略按 round 抽，整组同 round 才能整组抽

**算法：**

```python
import math
from collections import defaultdict

def unify_group_round(batch_questions):
    """4b round 阶段末尾必跑：同 group_id 的子题统一 round = ceil(平均)"""
    groups = defaultdict(list)
    for q in batch_questions:
        if q.get("group_id"):
            groups[q["group_id"]].append(q)
        # 单题无 group_id 不处理

    for gid, qs in groups.items():
        if len(qs) <= 1:
            continue  # 单子题（虽然有 group_id 但只 1 道）不处理
        rounds = [q["round"] for q in qs if q.get("round") is not None]
        if not rounds:
            continue
        avg = sum(rounds) / len(rounds)
        unified_round = math.ceil(avg)
        # 写回每个子题
        for q in qs:
            q["_round_before_group_unify"] = q["round"]  # 留痕
            q["round"] = unified_round
```

**执行时机：**
- pipeline 4b round 阶段末尾（写回 batch round 字段之前）
- 在数学 V4 脚本输出后、写回 batch 之前调用
- 在语文 LLM 判 round 后、写回 batch 之前调用
- 在物理/化学自由判后、写回 batch 之前调用

**举例：**

| 子题 round | 平均 | ceil | 统一后 |
|---|---|---|---|
| R1+R2+R3 | 2.0 | 2 | R2 |
| R1+R2+R2 | 1.67 | 2 | R2 |
| R1+R1+R2 | 1.33 | 2 | R2 |
| R2+R3+R4 | 3.0 | 3 | R3 |
| R3+R4 | 3.5 | 4 | R4 |
| R1+R1 | 1.0 | 1 | R1 |
| R2+R2+R2 | 2.0 | 2 | R2 |

**留痕字段：** `_round_before_group_unify` 记原始判定值，便于将来 reviewer 反思"组内 round 差异大是否反映出题逻辑问题"。

**禁止例外：**
- ❌ "这组只有 1 道是 R3，其他都 R1，统一到 R2 太高了" → 仍按 ceil(1+1+1+3)/4 = 2.0 = ceil → R2

### 9.1 完整性优先（V3.12.5 强化）：要么完整入库，要么整题放弃

**原则：** 一道题入库时**必须包含原题中的所有信息**——题面、候选池、共同上文、阅读材料、附图、附表、附说明等一个不漏。**任何一项识别不清或残缺，整题跳过**，不要"部分入库 + 缺什么就丢什么"。

**触发整题跳过的条件（任一满足即放弃）：**
- OCR 失败 / 扫描质量差 / 关键文字模糊
- 嵌入私有字体导致 pdftotext 出乱码
- 附图/几何图模糊看不清
- 听力音频质量差
- **听录音衍生题**（即使有答案 A/B/C/D，但题面"这段故事""这段话"指向录音内容，无音频学生无法判断）→ 整题跳过（V3.12.22 A6）
- 阅读题原文缺失或被切断
- 候选池/共同上文残缺（参见 §5.4 系列组合题纪律）
- 任何"靠猜"才能补上的信息

**禁止做法：**
- ❌ "这道题文字看清了，图模糊就丢图入库" → 整题跳过
- ❌ "候选池没扫到，按语境硬填一个答案入库" → 整题跳过
- ❌ "答案配对模糊，但题目内容还能用，先入库再说" → 整题跳过
- ❌ "OCR 漏几个字，多模态/重绘补上" → 整题跳过
- ❌ "标 _unverified 等以后修" → 直接删除/不入库

**记录：** 跳过的题/卷写到 `manifest.skipped[]`，注明原因。**不要**浪费 token 抢救。

**理由：** 真题量足够（150+ 卷），找模糊题修复成本远高于换下一卷。残缺入库会污染题库（Famin V3.10 多模态幻觉伪题 122 道事件已是反面教材）。

详见 `feedback_unclear_abandon.md`（同一原则的记忆侧）。

### 9.1.docx 信息找回优先（V3.12.22 Issue 2 Famin 决策）

**docx 与 PDF 不同**，§9.1 的"不全即弃" 是 PDF 时代措施（私有字体 cid 编码 / 扫描型整卷不可救）。docx 时代**信息丢失要先想能不能找回**：

**找回机制（V3.12.22 起强制）：**

1. **OMath 段空白** → `extract_docx.py` Step 3.5 自动转 LaTeX → 读 `raw_with_omath.txt`（详见 SKILL.md ⊥3a + rules/images_docx.md 规则 4.5）
2. **表格 raw 拼接散乱** → markdown `| ... |` 重建（详见 §5.6）
3. **pmap 找不到图归属** → ±3 段窗口扫 `media/` 未归属图（详见 rules/images_docx.md 规则 5）
4. **答案算法不严谨** → `|||` 多答案兜底（详见 rules/no_doctoring.md）

**只在以下情况整题跳**（docx 时代严格收紧）：

- 信息无法找回（如几何图模糊到 worker 看不清 / OMath 转换异常 / 表格列丢失）
- 题目暂时不适合 app 作答 / 评判（画图题 / 开放题 / 纯主观题）

### 9.1.docx-skipped 跳过题记位置（V3.12.22 Issue 2）

**reason 词典**:
- `open_drawing_question`: 需作图（画对称/平移/坐标点）
- `subjective_essay`: 纯主观长答（作文/谈谈感想）

### 9.1.docx-rebundle 组合题部分跳过 → 剩余重组（V3.12.22 Issue 2）

- 跳 (2) → (1)(3) 剩两题：保留 `group_id`，`group_order` 重排为 1, 2
- 跳 (1)(2) → 只剩 (3)：单题入（**不组 group_id**）
- 全部跳：整题进 `_skipped_for_future`，不入库

**与原 ⊥1 区别**: 原 ⊥1 防"worker 简化大题只入第一小题"——动机是偷懒。本规则允许的"部分跳重组"动机是"不能作答的题不入"，全程透明记录。

```python
    if 图_无关键信息 or 仅装饰: return "skip_image"   # content 文字够，图丢
    if 图_不可识别 or 模糊: return "abandon"          # 走 §9.4 整题跳过

    # V3.12.17 收紧：理科（数学/物理/化学）默认 SVG，screenshot 仅当 SVG 实画不出
```

**判定责任：** 4a annotate agent 看 PDF 后**先尝试用 SVG 描述**，能描述就走 §9.3；尝试失败（描述需要超过 50 个 path command 或含手绘细节）才走 §9.2。不能凭空选 screenshot 偷懒。

### 9.2 清晰图片 → 必须截图嵌入（V3.12.5 强化为强制；V3.12.13 加精准切边硬约束）

**V3.12.13 截图硬约束（解决"截入下一题/重复文字"反复 bug）：**

| 反例（V3.12 实测） | 正例 |
|---|---|
| ❌ 截图把题面文字也截入（content 已写一遍）| ✅ 截图只截图形 bbox 区域，文字留 content |
| ❌ 截图把下一题题号截入（边界判定不严）| ✅ 用 pdfplumber 拿当前题号 / 下一题号 char bbox 精确切边 |
| ❌ 题号 + 整页全截 | ✅ 题号 y - 5 到 下一题号 y - 5 |
| ❌ 复杂图截糊（分辨率低）| ✅ 200 dpi 起 |

**强制 pipeline（V3.12.13 起）：**

```python
import pdfplumber
def crop_question_image(pdf_path, page_num, q_num, next_q_num):
    """精准切题号 q_num 对应的图（仅图形区域，不含文字）"""
    with pdfplumber.open(pdf_path) as pdf:
        page = pdf.pages[page_num]
        # 1. 找题号 char y 坐标
        cur_top = find_char_y(page.chars, f'{q_num}．') or find_char_y(page.chars, f'{q_num}.')
        next_top = find_char_y(page.chars, f'{next_q_num}．') or page.height
        # 2. 在 [cur_top, next_top] 范围内找 image / lines / curves bbox
        img_bbox = find_graphic_bbox(page, cur_top, next_top)
        if img_bbox is None: return None
        # 3. 裁剪（仅图形区，不含文字）
        return page.crop(img_bbox).to_image(resolution=200).original
```

工具：`tools/realpaper/crop_question.py`（V3.12.13 待实施）



**适用所有科目**（数学/语文/英语/物理/化学）：

> 凡题面提到"如图""下图""右图""示意图""图中""图所示""观察图""看图"等任意指示语，**必须**按 §9.2 / §9.3 之一处理。**禁止**入库时不带图。

#### 9.3.1 SVG 画图 4 步流程（V3.12.17 强制硬约束）

完整纪律详见记忆 `feedback_svg_drawing_discipline.md`。

**步骤 1: PDF crop 看原图（不许跳过）**

### 9.4 何时跳过整题（V3.12.5 严格收紧）

**仅在以下情况跳过整题，记入 `manifest.skipped[]`：**

- ✅ 扫描质量差，图本身模糊到看不清（PDF 模糊 / 嵌入私有字体 / OCR 后字图都丢失）
- ✅ 需要画图作答（学生题目要求"在方格纸上画出..."、"作 XX 的高..."等，无法用 app 输入法回答）
- ✅ 图复杂到截图也无意义（如电路连线图，数据全在线条颜色/位置上，截图后学生看不清）

**禁止理由（V3.12.5 起列入反例）：**

- ❌ "图复杂，重绘 SVG 不值得就跳过" —— 改走 §9.2 截图路径，截图比重绘便宜
- ❌ "agent 不会用 pdftoppm，跳过省事" —— 必须学会用 pdftoppm
- ❌ "title 提到图但题面文字够用，丢图入库" —— V3.12.5 §9.1 完整性原则禁止
- ❌ "条形/扇形/折线统计图复杂，跳过" —— 截图嵌入即可
- ❌ "立体几何图截图也看不清" —— 看具体题，多数 200dpi 渲染清晰可读
- ❌ "审核 token 贵跳过快" —— 缺图入库 = 残缺 = §9.1 违反

**反例（V3.12.5 实测）：**
V3.10 入库 6 卷 math 北师大单元卷，agent 默认走 §9.4 跳过，实际跳掉 5-8 道/卷的图依赖题，但其中 80%+ 是清晰条形图/示意图，本应走 §9.2 截图嵌入。1050 题 0 图就是反面教材。

### 9.5 入库 agent 调用纪律（V3.12.5 强制）

每次入库 agent 处理一卷时：

1. **预扫**：先用 `pdftoppm -png -r 100 file.pdf preview` 把全卷渲染成 PNG，肉眼粗看哪些题带图
2. **分类**：每道题按 §9.2 / §9.3 / §9.4 决策：
   - 清晰图（含表/统计图）→ §9.2 高清截图（200 dpi）+ 裁剪 + image_data 字段
   - 简单几何/坐标 → §9.3 SVG 重绘（agent 直接生成）
   - 真模糊 / 画图作答题 / 极复杂连线 → §9.4 跳过 + manifest.skipped 注明具体原因
3. **入库前自检**：扫 batch JSON 里所有 content 含"如图/下图/右图/示意图/图中/图所示/看图/观察图"等关键词的题，**必须**有 `image_data` 或 content 内嵌 `<svg>`，否则**整题撤回**重处理
4. **commit message** 报告：`图嵌入 N 道（image_data X / SVG Y），§9.4 跳过 Z 道（具体原因列出）`

```python
# 用 libreoffice headless 转纯文本，sha1 缓存
sha1 = hashlib.sha1(open(path, 'rb').read()).hexdigest()
cache_dir = f'.cache/realpaper/{sha1}'
if not os.path.exists(f'{cache_dir}/raw.txt'):
    subprocess.run(['libreoffice', '--headless', '--convert-to', 'txt', '--outdir', cache_dir, path])
return open(f'{cache_dir}/raw.txt', encoding='utf-8').read()
```

### Step 2: segment.py
正则切题：
```
re.split(r'(?:^|\n)\s*(?:[一二三四五六七八九十]+[、.])', text)  # 大题
re.split(r'(?:^|\n)\s*\d+\s*[.、]', text)                    # 小题
re.split(r'(?:^|\n)\s*\(\d+\)', text)                        # 子题
```

### Step 3: match_ans.py
```python
# 找答案段
ans_start = re.search(r'(参考答案|答案|Answer Key|【答案】)', text)
if ans_start:
    ans_text = text[ans_start.end():]
    # 题号映射：1. xxx 2. xxx
    answers = dict(re.findall(r'(\d+)[.\s]+([^\n]+)', ans_text))
```

## 12. 失败回滚 / 错误处理

| 错误类型 | 处理 |
|---|---|
| libreoffice 转换失败 | manifest.skipped + 等待手工处理 |
| OCR 扫描质量太差 | 整文件跳过 + manifest.skipped |
| 切题失败（题号识别不出）| 整文件跳过 + manifest.issues |
| 答案缺失 | Claude 求解 + explanation 标 `answer_source: claude_solved` |
| KP 不在清单 | 入 kp_pending.json |
| chapter 不在清单 | 这是 bug，必须修（不允许进等候区）|
| 题型识别歧义 | Claude 二选一 + explanation 注明 |
| 同 sha1 文件已处理 | 跳过整文件（manifest 校验）|
| 单 commit 题量过大（> 500）| 拆批 commit |
