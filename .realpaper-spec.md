## 2. 9 步 Pipeline（V3.12.5 起 round 判定与 annotate 分离）

```
真题文件（.doc/.docx/.pdf）
  └→ [1 extract]    libreoffice headless --convert-to txt → .cache/realpaper/<sha1>/raw.txt
  └→ [2 segment]    正则切题（"^一、""1.""(1)"等模式 + 大题套小题用 group_id）→ stems[]
  └→ [3 match-ans]  找"参考答案/答案"段；或独立答案文件配对
  └→ [4a annotate]  Claude 会话级标注：type / chapter (∈158) / KP (∈491) / answer / explanation
                    ⚠ round 字段强制留 null，不许 LLM 自由判
  └→ [4b round]     按科目当前阶段判 round（见 §2.1 三阶段表）：
                      - 阶段 0（无锚点）：LLM 自由推理判
                      - 阶段 1（有锚点+rubric）：LLM 必须读 anchors_*.json + rubric_*.md 判
                      - 阶段 2（skill/脚本成熟）：强制脚本（如数学 d2_math_review_v4.py）
  └→ [5 validate]   py 脚本校验 chapter/KP/题型一致性 + KP 不在清单的题入等候区
  └→ [6 emit]       batch JSON 双写 assets/data/batches/ + question_bank/
  └→ [7 register]   question_bank/index.json + main.dart _bundledBatchAssets
  └→ [8 commit]     git commit + push + 更新 manifest
```

每步详细见 §10 实施细则。

### 2.1 Round 判定三阶段（V3.12.5 起硬规则）

每科 round 判定按"渐进沉淀"演化，**绝不退化**（每次 Famin verify 反馈后必须立即沉淀，让下批扫描受益）。

#### 三阶段定义

| 阶段 | 触发条件 | round 判定方式 | Famin 角色 |
|---|---|---|---|
| **0 无锚点** | 该科目首次扫描 / 锚点尚未建立 | LLM 自由推理判 | 抽审 verify markdown（锚点候选 + 改档）|
| **1 锚点+rubric** | 首轮 verify 反馈完成 → 锚点 + rubric 文档已沉淀 | **LLM 必须读 `anchors_*.json` + `rubric_*.md` 判**（不许凭空判，必须引用锚点 ID）| 继续抽审，反馈累积 |
| **2 skill/脚本** | rubric 稳定 + 累计 ≥ 3-4 轮 verify + 75% 严格对齐 | 脚本强制（python tools/d2_*_review_v*.py）| 仅 flag_review（high_variance / anchor_disagree）抽审 |

#### 各科目当前阶段（2026-05-08）

| 科目 | 阶段 | 判定方式 | 文档资产 |
|---|---|---|---|
| **数学** | ✅ 阶段 2 | `tools/d2_math_review_v4.py` 强制 | `~/.claude/skills/difficulty-math/` (anchors+evaluate+rubric+scripts) |
| **语文** | ⏳ 阶段 0→1（首轮 verify 进行中）| 等 Famin 审完 → 升阶段 1 | `docs/anchor_questions_g6_chinese.json`（20 道未审）+ rubric 待 Famin 审完后沉淀 |
| **英语** | ✅ 阶段 2（特殊路径）| Cambridge composition skill 自带 round（A2/B1/B2/C1 → R1/R2/R3/R4）| `~/.claude/skills/composition-english/`（待建）|
| **物理** | ⏳ 阶段 0 | LLM 自由判 | 待首轮扫描后建立锚点 |
| **化学** | ⏳ 阶段 0 | LLM 自由判 | 待首轮扫描后建立锚点 |

#### 阶段升级触发流程（强制）

每次 Famin verify 反馈后**立即**执行：

```
verify markdown 收到反馈
  ↓
1. 锚点更新：把同意的锚点写入 docs/anchor_questions_<grade>_<subject>.json
              + _famin_review.comment 字段
  ↓
2. Rubric 沉淀：把 reasoning 抽提到 docs/rubric_<subject>.md（首次→新建）
              重点记录"什么档对应什么特征" + "Famin 推翻 LLM 的典型 case"
  ↓
3. 反向修订：revert 项 → 改 batch round 字段（双写）+ commit
  ↓
4. 阶段判定：
     - 首轮 verify 完成 → 阶段 0 → 阶段 1（下批扫描 prompt 必须引用锚点+rubric）
     - 累计 ≥ 3-4 轮 verify + 75% 严格对齐 → 阶段 1 → 阶段 2（生成 reviewer 脚本 + skill）
  ↓
5. 工作流文件同步：
     - .realpaper-spec.md §2.1 各科阶段表更新
     - feedback_round_judge.md 同步
     - 新建 skill 时把锚点+rubric+脚本打包到 ~/.claude/skills/difficulty-<subject>/
```

**审了不沉淀 = 白审**。这是 feedback_strategy_to_execution 的硬要求。

#### 禁止行为

- ❌ 入库 agent 在 4a 阶段直接给 round 字段赋值（哪怕"为方便 4b 跑"）
- ❌ 数学科 LLM 自由判 round（已阶段 2，必须跑脚本）
- ❌ 已进入阶段 1 的科目，agent prompt 不引用锚点+rubric 凭空 LLM 判（这是退化）
- ❌ 跳过 verify 直接把 LLM 判的 round 落库（V3.12.5 commit 161072d 反面教材）
- ❌ Famin verify 完不沉淀就启动下批扫描（必须先沉淀再扫）

## 5. 题型识别规则

| 模式 | 判定 type |
|---|---|
| 题面含 4 行 `A./B./C./D.` | choice |
| 题面是单一陈述句 + `(    )` 或 `判断对错`（"对的画√错的画×"等指引）| **judgment**（V3.10 新）|
| 题面含 `_____` 或 `（  ）`且无 ABCD 选项；答案 ≤ 8 字 | fill |
| 题面含 `计算下列` `解方程` `列式` `求...的值` | calculation |
| 题面含 `作文` `根据材料写` `谈谈你的看法` `写一篇` `不少于 X 字` `谈谈感受` | **subjective**（保留区，source 加 `_subj_held`）|

### 5.1 输入法限制白名单（feedback_question_quality.md 继承）
fill 答案必须满足：
- 中文：单字 / ≤4 字短词 / ≤5 字古诗原文 / 修辞名词 / 作家作品名
- 数字：纯数字 / 简单分数（"1/3"）/ 简单比例（"3:1"）
- 不允许：长句 / 整段表达 / 含特殊符号（π/²/×/拼音声调等 → 改 choice）

### 5.2 主观题归类
- 语文作文/简答/论述 → subjective + `_subj_held`（家长批改机制 V3.8.3 已建，但暂不启用，等 AI API）
- 政治/历史的简答（不在范围）→ 跳过整题
- 数学/物理/化学的简答（如"简述实验步骤"）→ subjective + `_subj_held`

### 5.3 判断题（V3.10 新）
- type=`judgment`；answer 必须是 `"对"` 或 `"错"`（单字）
- AnswerMatcher 接受等价写法：对/正确/√/T/true/yes 视为"对"；错/错误/×/F/false/no 视为"错"
- options 字段填 `null`（UI 自动渲染对/错二选一按钮）
- alt_answers 不需要（matcher 已内置归一化）

### 5.4 复合题：先 fill 后 choice 拆分（V3.12.5 起强制）

**适用结构：** 大题先给一组待补全的词/句（A 部分），再用补全后的词去填语境空（B 部分）。典型如部编六下 d1 单元卷 1 第 4 大题：

```
4. 把下列四字词语补充完整，并选词填空。
   万(  )(  )新   (  )(  )不同   万不(  )(  )
   (  )灯(  )彩  (  )(  )分明   两(  )三(  )
   (1) 春节到了，中华大地上到处____，显出一派____的气象。
   (2) 在外边做事的人，除非____，必定赶回家来吃团圆饭。
   (3) 少数民族和汉族的饮食风俗虽然____，但各族人民热爱祖国、热爱民族大团结的心是相同的。
```

**禁止做法：**
- ❌ 把 (1)(2)(3) 直接当独立 fill 题入库（候选池上下文丢失，学生看不到 6 候选成语，被迫凭语境想 → 可能写错形或同义不等价答）
- ❌ 把整道大题当一个 fill 题（无法判分多空答案）

**强制做法：A 部分→fill 题群；B 部分→choice 题群（候选池 = A 部分补全后的词）**

```yaml
A 部分（补全题）:
  type: fill
  N 道独立 fill（每个待补全的词一题）
  group_id: <大题 id>_a
  group_order: 1..N
  content: "把四字词语补充完整：万(  )(  )新"
  answer: "象更"   # 或 "象,更"（多空逗号分隔）

B 部分（选词填空）:
  type: choice
  M 道独立 choice（每个语境空一题）
  group_id: <大题 id>_b
  group_order: 1..M
  content 必须把 A 部分补全后的所有候选词列出 + 当前语境句:
    "从下列成语中选填到空中（每个成语只用一次）：万象更新 / 截然不同 / 万不得已 /
     张灯结彩 / 黑白分明 / 两面三刀。
     语境：春节到了，中华大地上到处(    )，显出一派___的气象。本题选第 1 个空。"
  options: ["万象更新", "截然不同", "万不得已", "张灯结彩", "黑白分明", "两面三刀"]
  answer: "张灯结彩"
```

**理由：**
1. A 部分考"字面识记"（成语字形）→ 适合 fill，每空独立判
2. B 部分考"语境匹配"（候选池辨析）→ 适合 choice，候选池显式呈现给学生
3. 拆开后两部分都能被 AnswerMatcher 准确判错，且符合 §5.1 输入法限制
4. group_id 维持原大题关联，错题集仍能按"大题"聚合复盘

**多语境空的处理：**
- 一句含 N 个空（如"到处___，显出一派___"）→ 拆 N 道 choice，content 标"本题选第 K 个空"
- 候选池可重复使用 → options 列出全部 6 个候选；候选池一次性消耗 → content 注明"每个成语只用一次"

**小变体：**
- A 部分极少（≤2 道）→ 仍拆，不要合并
- A 部分 = 课内默写（如"补全古诗：少壮不努力，___"）+ B 部分语境填 → A 仍 fill；B 候选池 = A 答案集

**判定 round（与本节配合）：**
- A 部分 fill：按"字面识记"，多数 R1
- B 部分 choice：按"候选池辨析"难度判（候选池数量 + 语境隐蔽度），一般 R2，候选数 ≤ 3 且语境直白可降 R1

**实施位置：** Pipeline §2 第 2 步 `segment.py` 必须识别"...补充完整，并选词填空"或"...选择恰当的成语填入下列各句"等大题指令，按本节拆。无法判定时整道入"等候区"等人工审核（不要硬拆）。

## 6. 难度档（round）判定

由 Claude 主观判，基于 3 个维度：

### 6.1 试卷类型基线

| 试卷类型 | 整卷 round 基线 |
|---|---|
| 单元/小测 | R1 |
| 月考 | R1-R2 |
| 期中/期末 | R2-R3 |
| 中考真题 | R2-R4 全 |
| 培优/竞赛 | R3-R4 |

### 6.2 卷内位置加权
- 卷首选择前 1/3：基线 -1（最低 R1）
- 卷面中段：基线
- 卷尾大题/压轴：基线 +1（最高 R4）

### 6.3 KP 复杂度
- 单 KP 题：保持上述判定
- 跨年级综合题：**取最高年级 KP**（强制规则）
  - 7 + 9 年级 KP → 归 9 年级 KP
  - 7 + 8 + 9 → 归 9 年级 KP

### 6.4 round 含义（V3.8 沿用）
- R1：基础（教材内单一 KP 直接套用）
- R2：中等（同章节多 KP 综合）
- R3：较难（跨章节综合 / 应用题情境）
- R4：竞赛/压轴（跨年级综合 / 创新探究 / 拓展深度）

## 10. 实施细则

### Step 1: extract.py
```python
# 用 libreoffice headless 转纯文本，sha1 缓存
sha1 = hashlib.sha1(open(path, 'rb').read()).hexdigest()
cache_dir = f'.cache/realpaper/{sha1}'
if not os.path.exists(f'{cache_dir}/raw.txt'):
    subprocess.run(['libreoffice', '--headless', '--convert-to', 'txt', '--outdir', cache_dir, path])
return open(f'{cache_dir}/raw.txt', encoding='utf-8').read()
```

### Step 2: segment.py
正则切题：
```
re.split(r'(?:^|\n)\s*(?:[一二三四五六七八九十]+[、.])', text)  # 大题
re.split(r'(?:^|\n)\s*\d+\s*[.、]', text)                    # 小题
re.split(r'(?:^|\n)\s*\(\d+\)', text)                        # 子题
```

### Step 3: match_ans.py
```python
# 找答案段
ans_start = re.search(r'(参考答案|答案|Answer Key|【答案】)', text)
if ans_start:
    ans_text = text[ans_start.end():]
    # 题号映射：1. xxx 2. xxx
    answers = dict(re.findall(r'(\d+)[.\s]+([^\n]+)', ans_text))
```

### Step 4: annotate（Claude inference）
对每题，Claude 输出 JSON：
```json
{
  "stem_idx": 5,
  "type": "choice",
  "chapter": "圆柱与圆锥",
  "knowledge_point": "圆柱与圆锥/圆柱体积",
  "round": 2,
  "explanation": "..."
}
```

### Step 5: validate.py
检查：
- chapter ∈ curriculum_seed（按 subject + grade 过滤）
- knowledge_point ∈ knowledge_points_seed（按 subject 过滤）
- type 一致（题面有 ABCD ↔ type=choice）
- 不在清单的 KP → 转 kp_pending.json

### Step 6: emit batch JSON
```json
{
  "source": "realpaper_g6_math_beijing_pinggu_qimo_001",
  "subject": "math",
  "grade": 6,
  "knowledge_points_added": [],
  "questions": [...]
}
```

双写到：
- `assets/data/batches/<source>.json`
- `question_bank/<source>.json`

### Step 7: register
- `question_bank/index.json` 加 batches[] 项 + version+1
- `lib/main.dart` `_bundledBatchAssets` 追加 `'assets/data/batches/<source>.json'`

### Step 8: commit
- `git add` 上述所有改动
- `git commit -m "Realpaper batch: <region> <grade> <subject> <type>"`
- `git push origin main`
- 更新 `docs/realpaper_manifest.json`

## 12. 失败回滚 / 错误处理

| 错误类型 | 处理 |
|---|---|
| libreoffice 转换失败 | manifest.skipped + 等待手工处理 |
| OCR 扫描质量太差 | 整文件跳过 + manifest.skipped |
| 切题失败（题号识别不出）| 整文件跳过 + manifest.issues |
| 答案缺失 | Claude 求解 + explanation 标 `answer_source: claude_solved` |
| KP 不在清单 | 入 kp_pending.json |
| chapter 不在清单 | 这是 bug，必须修（不允许进等候区）|
| 题型识别歧义 | Claude 二选一 + explanation 注明 |
| 同 sha1 文件已处理 | 跳过整文件（manifest 校验）|
| 单 commit 题量过大（> 500）| 拆批 commit |
