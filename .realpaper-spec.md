## 2. 9 步 Pipeline（V3.12.5 起 round 判定与 annotate 分离）

```
真题文件（.doc/.docx/.pdf）
  └→ [1 extract]    libreoffice headless --convert-to txt → .cache/realpaper/<sha1>/raw.txt
  └→ [2 segment]    正则切题（"^一、""1.""(1)"等模式 + 大题套小题用 group_id）→ stems[]
  └→ [3 match-ans]  找"参考答案/答案"段；或独立答案文件配对
  └→ [4a annotate]  Claude 会话级标注：type / chapter (∈158) / KP (∈491) / answer / explanation
                    ⚠ round 字段强制留 null，不许 LLM 自由判
  └→ [4b round]     按科目分支判 round：
                      - 数学：python tools/d2_math_review_v4.py（V4 脚本判，结果写入 round）
                      - 其他科目：LLM 自由推理判 + 单独 verify markdown 给 Famin 抽审
  └→ [5 validate]   py 脚本校验 chapter/KP/题型一致性 + KP 不在清单的题入等候区
  └→ [6 emit]       batch JSON 双写 assets/data/batches/ + question_bank/
  └→ [7 register]   question_bank/index.json + main.dart _bundledBatchAssets
  └→ [8 commit]     git commit + push + 更新 manifest
```

每步详细见 §10 实施细则。

### 2.1 Round 判定纪律（V3.12.5 起硬规则）

| 科目 | 当前阶段 | round 判定方式 | 是否需 Famin verify |
|---|---|---|---|
| **数学** | ✅ 算法成熟（V4 75% 严格对齐）| `tools/d2_math_review_v4.py` 直接落 round | ❌ 默认信任，仅 flag_review（high_variance / anchor_disagree）抽审 |
| **语文** | ⏳ 算法未沉淀 | LLM 自由推理判 round | ✅ 抽审 markdown verify（D1 锚点 + 改档抽样）|
| **英语** | ✅ Cambridge KP 体系 + composition skill | composition skill 出题时已带 round（A2/B1/B2/C1 → R1/R2/R3/R4）| ❌ 体系内自校 |
| **物理** | ⏳ 算法未沉淀 | LLM 自由推理判 round | ✅ 抽审 markdown verify |
| **化学** | ⏳ 算法未沉淀 | LLM 自由推理判 round | ✅ 抽审 markdown verify |

**升级条件**：每科累计 Famin 抽审反馈 ≥ 4 轮 + 算法 75% 严格对齐 → 切到"脚本判 + 不审"。流程仿数学 D1→D2→V1→V2→V3→V4。

**禁止行为**：
- ❌ 入库 agent 在 4a 阶段直接给 round 字段赋值（哪怕"为方便 V4 跑"）
- ❌ 数学科 LLM 自由判 round（必须跑脚本）
- ❌ 跳过 verify 直接把 LLM 判的 round 落库（语文/物理/化学这次 161072d 跳步是反面教材）

## 3. chapter 命名规则

### 3.1 数学/物理/化学：教材章节（保持现状）
严格映射 `lib/database/curriculum_seed.dart` 现有数学 40 / 物理 22 / 化学 11 chapter（按北师大/人教教材单元编排）。

### 3.2 语文：按 KP 一级 category
chapter = KP fullPath 的第一段（13 类）：`古诗文` / `现代文阅读` / `写作` / `字词` / `文学常识` / `名著阅读` / `修辞` / `综合性学习` / `句式与标点` / `语法` / `语法与句式` / `课文与名著` / `阅读理解`。

### 3.3 综合卷（小升初/中考）

| chapter | 适用 |
|---|---|
| `小升初综合` | grade=6 数学/语文 真题（覆盖小学全程内容）|
| `中考综合` | grade=9 数学/语文/物理/化学 真题（覆盖初一到初三）|

V3.10 已扩 6 个综合 chapter（curriculum_seed.dart）。

### 3.4 一卷归类策略

- **小升初真题** → grade=6, chapter="小升初综合", KP=题主考内容（可跨小学全年级）
- **中考真题** → grade=9, chapter="中考综合", KP=题主考内容（可跨初一到初三）
- **单元/月考/期末真题** → grade=对应年级, chapter=对应年级 curriculum_seed 章节, KP=题主考内容

文件名信息（地区/年份/类型）写到 `source` 字段，**不进 chapter**。

## 4. source 命名规则（每卷唯一）

```
realpaper_g{6-9}_<subject_pinyin>_<region_pinyin>_<paper_type>_<seq>
```

例：
- `realpaper_g6_math_beijing_pinggu_qimo_001`
- `realpaper_g9_chem_2018_zhongkao_shanghai_001`
- `realpaper_g6_chinese_xiaoshengchu_moni_school1_001`

`<seq>` 防同地区同类型多卷冲突。

### 4.1 主观题保留区
常规 source 基础上加 `_subj_held` 后缀：
- 例：`realpaper_g6_chinese_beijing_qimo_001_subj_held`
- QuestionDao 抽题 SQL 已加 `source NOT LIKE '%_subj_held'` 过滤
- 等接入 AI 评分 API 后批量去后缀启用

### 4.2 翻译为英文的中文题（V3.11+ 启用）
原中文题 source 加 `_translated_en` 后缀（暂不抽到，避免重复练）。

## 5. 题型识别规则

| 模式 | 判定 type |
|---|---|
| 题面含 4 行 `A./B./C./D.` | choice |
| 题面是单一陈述句 + `(    )` 或 `判断对错`（"对的画√错的画×"等指引）| **judgment**（V3.10 新）|
| 题面含 `_____` 或 `（  ）`且无 ABCD 选项；答案 ≤ 8 字 | fill |
| 题面含 `计算下列` `解方程` `列式` `求...的值` | calculation |
| 题面含 `作文` `根据材料写` `谈谈你的看法` `写一篇` `不少于 X 字` `谈谈感受` | **subjective**（保留区，source 加 `_subj_held`）|

### 5.1 输入法限制白名单（feedback_question_quality.md 继承）
fill 答案必须满足：
- 中文：单字 / ≤4 字短词 / ≤5 字古诗原文 / 修辞名词 / 作家作品名
- 数字：纯数字 / 简单分数（"1/3"）/ 简单比例（"3:1"）
- 不允许：长句 / 整段表达 / 含特殊符号（π/²/×/拼音声调等 → 改 choice）

### 5.2 主观题归类
- 语文作文/简答/论述 → subjective + `_subj_held`（家长批改机制 V3.8.3 已建，但暂不启用，等 AI API）
- 政治/历史的简答（不在范围）→ 跳过整题
- 数学/物理/化学的简答（如"简述实验步骤"）→ subjective + `_subj_held`

### 5.3 判断题（V3.10 新）
- type=`judgment`；answer 必须是 `"对"` 或 `"错"`（单字）
- AnswerMatcher 接受等价写法：对/正确/√/T/true/yes 视为"对"；错/错误/×/F/false/no 视为"错"
- options 字段填 `null`（UI 自动渲染对/错二选一按钮）
- alt_answers 不需要（matcher 已内置归一化）

## 6. 难度档（round）判定

由 Claude 主观判，基于 3 个维度：

### 6.1 试卷类型基线

| 试卷类型 | 整卷 round 基线 |
|---|---|
| 单元/小测 | R1 |
| 月考 | R1-R2 |
| 期中/期末 | R2-R3 |
| 中考真题 | R2-R4 全 |
| 培优/竞赛 | R3-R4 |

### 6.2 卷内位置加权
- 卷首选择前 1/3：基线 -1（最低 R1）
- 卷面中段：基线
- 卷尾大题/压轴：基线 +1（最高 R4）

### 6.3 KP 复杂度
- 单 KP 题：保持上述判定
- 跨年级综合题：**取最高年级 KP**（强制规则）
  - 7 + 9 年级 KP → 归 9 年级 KP
  - 7 + 8 + 9 → 归 9 年级 KP

### 6.4 round 含义（V3.8 沿用）
- R1：基础（教材内单一 KP 直接套用）
- R2：中等（同章节多 KP 综合）
- R3：较难（跨章节综合 / 应用题情境）
- R4：竞赛/压轴（跨年级综合 / 创新探究 / 拓展深度）

## 7. KP 严格匹配 + 等候区机制

### 7.1 严格匹配（feedback_kp_discipline.md 继承）
题的 `knowledge_point` 字段必须**严格**等于 `lib/database/knowledge_points_seed.dart` 的 fullPath（区分大小写、空格、括号）。

**禁止**自创变种 / 自动 add 新 KP。

### 7.2 等候区机制（V3.10 新）
不在 491 KP 清单的题：
- **不入库**
- 写到 `docs/realpaper_kp_pending.json`
- 含字段：source_file / batch_source / question_data（完整题对象 + KP=null）/ claude_suggestion (`candidate_kp` + `action: new_kp/merge_to/drop` + `merge_target` + `reason`)

### 7.3 每科 10 卷 review
我累计该科 10 卷的 `kp_pending` → 按候选 KP 聚类 → 给 Famin 看：
- 候选 KP 名
- 涉及题数
- 示例题面 2-3 个
- 我的建议（merge_to / new_kp / drop）
- 现有相近 KP（如有）

Famin 三选一：
1. **合并** → 题改 KP 后入 `realpaper_g{N}_{科}_kp_supplement` batch
2. **新建** → 先扩 `knowledge_points_seed.dart`，再入 supplement batch
3. **放弃** → 删等候区记录

## 8. 答案匹配

按优先级：

1. **同文件含答案段**（~52% 文件名带"含答案"/"试题及答案"）
   - 在 raw.txt 里找 `参考答案|答案|Answer Key` 之后段
   - 题号映射（"1." → 答案 "1." 后内容）

2. **独立答案文件**（同目录下 `*答案*` `*解析*` 同名文件）
   - 配对加载 + 题号对齐

3. **完全无答案**
   - Claude 会话级求解
   - 标记 `answer_source: claude_solved` 写入 explanation 末尾

## 9. 图片/质量纪律（Famin 决策）

### 9.1 图/文字不清晰 → 直接放弃
- OCR 失败 / 扫描质量差 / 关键文字模糊 → 整题跳过
- 记入 `manifest.skipped[]`，注明原因
- **不要浪费 token 修复**

### 9.2 清晰图片 → 直接截图嵌入
- 用 pdftoppm/pdf2image 截图 → base64 → image_data 字段（V3.7.6 schema 支持）
- 限制：base64 图 ≤ 200KB（避免 batch JSON 爆炸）

### 9.3 需要重绘的图 → 重绘
- 数学几何图 → SVG（V3.7.9 渲染支持）
- 物理电路图 → SVG
- 化学分子结构 → SVG 或 LaTeX
- 化学方程式 → LaTeX `$...$`（V3.7.9 已支持）
- 数学公式 → LaTeX `$...$`

### 9.4 重绘成本评估
- 简单图（直角坐标、几何方位）→ Claude 直接生成 SVG
- 复杂图（电路连线、立体几何）→ 评估时间 / 复杂度 → 不值得就跳过

## 10. 实施细则

### Step 1: extract.py
```python
# 用 libreoffice headless 转纯文本，sha1 缓存
sha1 = hashlib.sha1(open(path, 'rb').read()).hexdigest()
cache_dir = f'.cache/realpaper/{sha1}'
if not os.path.exists(f'{cache_dir}/raw.txt'):
    subprocess.run(['libreoffice', '--headless', '--convert-to', 'txt', '--outdir', cache_dir, path])
return open(f'{cache_dir}/raw.txt', encoding='utf-8').read()
```

### Step 2: segment.py
正则切题：
```
re.split(r'(?:^|\n)\s*(?:[一二三四五六七八九十]+[、.])', text)  # 大题
re.split(r'(?:^|\n)\s*\d+\s*[.、]', text)                    # 小题
re.split(r'(?:^|\n)\s*\(\d+\)', text)                        # 子题
```

### Step 3: match_ans.py
```python
# 找答案段
ans_start = re.search(r'(参考答案|答案|Answer Key|【答案】)', text)
if ans_start:
    ans_text = text[ans_start.end():]
    # 题号映射：1. xxx 2. xxx
    answers = dict(re.findall(r'(\d+)[.\s]+([^\n]+)', ans_text))
```

### Step 4: annotate（Claude inference）
对每题，Claude 输出 JSON：
```json
{
  "stem_idx": 5,
  "type": "choice",
  "chapter": "圆柱与圆锥",
  "knowledge_point": "圆柱与圆锥/圆柱体积",
  "round": 2,
  "explanation": "..."
}
```

### Step 5: validate.py
检查：
- chapter ∈ curriculum_seed（按 subject + grade 过滤）
- knowledge_point ∈ knowledge_points_seed（按 subject 过滤）
- type 一致（题面有 ABCD ↔ type=choice）
- 不在清单的 KP → 转 kp_pending.json

### Step 6: emit batch JSON
```json
{
  "source": "realpaper_g6_math_beijing_pinggu_qimo_001",
  "subject": "math",
  "grade": 6,
  "knowledge_points_added": [],
  "questions": [...]
}
```

双写到：
- `assets/data/batches/<source>.json`
- `question_bank/<source>.json`

### Step 7: register
- `question_bank/index.json` 加 batches[] 项 + version+1
- `lib/main.dart` `_bundledBatchAssets` 追加 `'assets/data/batches/<source>.json'`

### Step 8: commit
- `git add` 上述所有改动
- `git commit -m "Realpaper batch: <region> <grade> <subject> <type>"`
- `git push origin main`
- 更新 `docs/realpaper_manifest.json`

## 12. 失败回滚 / 错误处理

| 错误类型 | 处理 |
|---|---|
| libreoffice 转换失败 | manifest.skipped + 等待手工处理 |
| OCR 扫描质量太差 | 整文件跳过 + manifest.skipped |
| 切题失败（题号识别不出）| 整文件跳过 + manifest.issues |
| 答案缺失 | Claude 求解 + explanation 标 `answer_source: claude_solved` |
| KP 不在清单 | 入 kp_pending.json |
| chapter 不在清单 | 这是 bug，必须修（不允许进等候区）|
| 题型识别歧义 | Claude 二选一 + explanation 注明 |
| 同 sha1 文件已处理 | 跳过整文件（manifest 校验）|
| 单 commit 题量过大（> 500）| 拆批 commit |
