# Daughter Learning App — Cron 自动出题规范

**作用：** 这份文件是 Cron 远程 agent 每日生成题包的**唯一权威指引**。Cron agent 启动时**必须先读这个文件**，按规则自包含执行，不依赖任何本地 memory。

**如有冲突：** 以本文件为准。如对内容有疑问，提交一个 `cron-issue.md` 而不是猜测执行。

---

- 仓库：[famin1850618/daughter-learning-app](https://github.com/famin1850618/daughter-learning-app)（公开）

## 3. 文件路径与命名

**JSON 文件命名**：
```
batch_YYYY_MM_DD_g{6|7|8|9}_{math|chinese|english|physics|chemistry}_r{1|2|3|4}.json
```

例：`batch_2026_05_08_g7_math_r1.json`

**输出位置**（每个 batch 写两份）：
1. `assets/data/batches/<filename>.json` — APK 内置 bundle
2. `question_bank/<filename>.json` — jsDelivr CDN 路由

**注册步骤**（生成新 batch 后必做）：
1. 更新 `question_bank/index.json` 加入新 batch 元信息（source / url / subject / grade / count）
2. 更新 `lib/main.dart` 的 `_bundledBatchAssets` 数组加入新 asset 路径

## 4. JSON Schema

每个 batch 文件结构：

```json
{
  "source": "batch_2026_05_08_g7_math_r1",
  "subject": "math",
  "grade": 7,
  "knowledge_points_added": [
    {"category": "...", "name": "..."}
  ],
  "questions": [
    {
      "chapter": "<must match curriculum_seed.dart>",
      "knowledge_point": "<category>/<name>",
      "content": "题面文本，数学公式用 $...$ 包",
      "type": "choice | fill | calculation",
      "difficulty": "easy | medium | hard",
      "round": 1,
      "options": ["A. ...", "B. ...", "C. ...", "D. ..."],
      "answer": "B  或  '答案1|||答案2|||...'（多写法用 ||| 分隔）",
      "explanation": "解析",
      "image": "<svg ...>...</svg>  或 null",
      "audio_text": "听力题原文（仅英语）  或 null"
    }
  ]
}
```

**字段说明**：
- `subject`：Dart enum 标识符（小写英文）：`chinese / math / english / physics / chemistry`
- `grade`：6/7/8/9 整数
- `chapter`：必须严格匹配 `lib/database/curriculum_seed.dart` 中该年级该科目的章节名
- `knowledge_point`：`<category>/<name>` 路径形式，必须从 `lib/database/knowledge_points_seed.dart` 选；新批次引入的新 KP 通过 `knowledge_points_added` 字段一并入库
- `type`：仅这三个字符串
- `difficulty`：题目本身的复杂度（与 round 不同维度，round 是批次档）
- `round`：**必填** 1-4
- `options`：仅 type=choice 时填，4 项 `A./B./C./D.` 前缀
- `answer`：choice 题填字母（A/B/C/D）；fill/calculation 填答案文本，公式题用 `|||` 分隔等价写法
- `image`：SVG 字符串（以 `<svg` 开头）或 base64 data URL；几何/电路图用
- `audio_text`：英语听力朗读原文

## 5. KP 纪律（最关键）

**强制流程：每生成新批次前**：
1. 读 `lib/database/knowledge_points_seed.dart` 确认目标 (subject, grade) 的 KP 清单存在
2. **不存在则先列**：约 25-50 个二级 KP，按一级类目组织
3. 题目 `knowledge_point` 字段必须从清单中选，不允许自由填
4. 命名风格：`<一级类目>/<二级具体名>`（如 `比和比例/化简比`）
5. 同一概念跨年级用同名 KP（如 `修辞/比喻` 在六七八九通用）

**新 KP 通过 batch JSON 的 `knowledge_points_added` 字段同步入库**：app 端导入时合并到 `knowledge_points` 表。

## 6. 题目质量纪律

### 输入法限制（含这些字符的填空题必须改选择题）

- 数学符号：`π`、`²`/`³` 上下标、`×÷`、复杂分式（`1/3πr²h`）、根号 `√`
- 拼音声调：`á à ǎ ē é ě è` 等
- 希腊字母（除 π）：`α β γ θ φ`
- 化学下标：`H₂O`、`CO₂`
- 任何需切换 / 复制粘贴才能输入的符号

### 填空题只允许的答案类型

- 纯数字：`62.8`、`40`、`3.14`
- 简单分数：`1/3`、`2/3`
- 简单比例：`3:1`、`2:3`
- 单字 / 短词（≤4 字）
- 英文短词（≤8 字母）

**含限制项的题必须重写为 choice 4 选项。**

### 禁开放题

不收"描写__应该写什么"、"举例说明..."这类无唯一答案题。

### 数学公式 LaTeX

content 和 explanation 中数学表达**必须用 `$...$` 包 LaTeX**：
- 分数：`$\frac{1}{2}$`
- 上标：`$r^2$`
- 希腊字母：`$\pi$`
- 乘除：`$\times$` `$\div$`

例：`化简 $\frac{1}{2}:\frac{1}{4}$ = ?`

### 公式题 alt_answers

涉及多种等价写法的公式题（如圆锥体积），`answer` 字段用 `|||` 分隔：
```
"answer": "πr²h/3|||pi*r*r*h/3|||pi*r^2*h/3|||(1/3)*π*r²*h"
```

UI 显示用第一段（`displayAnswer = answer.split('|||').first`）。

## 7. 难度档 Rubric

每题必须按 round 分档，标准如下。

### 数学

- **R1 基础**：单一公式 / 概念识别 / 1 步运算（如"圆柱体积公式"、"求 12÷3"）
- **R2 中等**：2-3 步综合 / 单位换算 / 一般应用题
- **R3 较难**：多概念综合 / 几何代数交叉 / 实际情境建模
- **R4 竞赛**：思维拓展 / 多解 / 反证 / 数论拓展（仍贴合教材范围，但要思考路径）

### 语文

- **R1**：字音字形 / 词义辨析 / 修辞识别 / 课文事实
- **R2**：句意理解 / 短篇阅读基础题 / 修辞应用 / 文学常识
- **R3**：长篇阅读 / 古诗鉴赏 / 文言翻译 / 综合分析
- **R4**：经典深度赏析 / 跨篇比较 / 文化典故拓展

### 英语

- **R1**：基础词汇 / 时态简单变形 / 简单句型
- **R2**：句型转换 / 复合时态 / 中等阅读
- **R3**：长篇阅读 / 写作 / 语境推断 / 高频固定搭配
- **R4**：高阶语法 / 深度写作框架 / 同义辨析

### 物理

- **R1**：概念辨析 / 单一公式套用
- **R2**：2 步公式综合 / 实验现象分析
- **R3**：复合运动 / 实验设计 / 多公式综合
- **R4**：综合实验 / 复杂场景 / 公式推导

### 化学

- **R1**：元素符号 / 基础反应识别
- **R2**：基本方程式书写 / 简单计算
- **R3**：综合方程 / 多步推断
- **R4**：实验设计 / 综合工业应用 / 综合推断题

## 8. 题型 / 难度 分布建议（每科 200 题）

- **题型**：80 选 + 80 填 + 40 算（数学/物理/化学）；120 选 + 80 填（语文/英语，无 calc）
- **difficulty**：80 easy + 80 medium + 40 hard（与 round 正交）

## 9. 工作流（每次 cron 启动后必做）

1. **解析当前日期**：用 Bash `date -u +%Y-%m-%d` 拿 UTC 日期；北京时间 = UTC+8
2. **查阶段日程**：本文件第 2 节，找今天对应任务（年级 / 科目 / round）
3. **读 KP 清单**：`lib/database/knowledge_points_seed.dart`，确认目标 KP 存在；不存在先列
4. **并行生成各科**：用 Agent 并行（每科一个 Agent），每 Agent 生成 200 题
   - 严格遵守纪律：输入法限制 / LaTeX / KP 严格匹配 / round 标记
5. **写文件**：每个 batch 双写
   - `assets/data/batches/<filename>.json`
   - `question_bank/<filename>.json`
6. **注册新 batch**：
   - 更新 `question_bank/index.json` 加入条目
   - 更新 `lib/main.dart` 的 `_bundledBatchAssets` 加入路径
7. **flutter analyze**（确保无 error）
8. **commit + push**：commit message 格式 `Cron YYYY-MM-DD: <年级 R<round>> <科目列表> 各 200 题`
9. **回报**：在 issue tracker / 提交一个简短日志（commit hash + 总题数）

## 10. 第二/三阶段错题反馈（5/20 起）

阶段 2/3 在第 4 步前**额外**做：

1. clone 私有 repo `daughter-learning-data`
2. 找 `learning_data/{any-deviceName}/*.json` 最近 7 天文件
3. 聚合所有用户的 `by_kp` 段：按 `(subject, knowledge_point)` 求和 attempts/errors，重算 error_rate
4. 仅纳入 `attempts ≥ 3` 的 KP
5. 30% 反馈池题数按 error_rate 加权分配到这些 KP（生成相似变体题，按错题原 round 出）
6. 70% 池继续按章节循环铺新内容

## 11. 错误处理

- 单科生成失败 → 跳过该科，继续其他科目；commit 时只 push 成功的
- KP 清单缺失 → 立即先生成清单 + commit + 接着生成题包
- flutter analyze 报 error → 不 push，写 issue 报告
- repo 拉取失败 → 重试 3 次，仍失败则报警
