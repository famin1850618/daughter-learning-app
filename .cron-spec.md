## 3. 文件路径与命名

**JSON 文件命名**：
```
batch_YYYY_MM_DD_g{6|7|8|9}_{math|chinese|english|physics|chemistry}_r{1|2|3|4}.json
```

例：`batch_2026_05_08_g7_math_r1.json`

**输出位置**（每个 batch 写两份）：
1. `assets/data/batches/<filename>.json` — APK 内置 bundle
2. `question_bank/<filename>.json` — jsDelivr CDN 路由

**注册步骤**（生成新 batch 后必做）：
1. 更新 `question_bank/index.json` 加入新 batch 元信息（source / url / subject / grade / count）
2. 更新 `lib/main.dart` 的 `_bundledBatchAssets` 数组加入新 asset 路径

## 4. JSON Schema

每个 batch 文件结构：

```json
{
  "source": "batch_2026_05_08_g7_math_r1",
  "subject": "math",
  "grade": 7,
  "knowledge_points_added": [
    {"category": "...", "name": "..."}
  ],
  "questions": [
    {
      "chapter": "<must match curriculum_seed.dart>",
      "knowledge_point": "<category>/<name>",
      "content": "题面文本，数学公式用 $...$ 包",
      "type": "choice | fill | calculation",
      "difficulty": "easy | medium | hard",
      "round": 1,
      "options": ["A. ...", "B. ...", "C. ...", "D. ..."],
      "answer": "B  或  '答案1|||答案2|||...'（多写法用 ||| 分隔）",
      "explanation": "解析",
      "image": "<svg ...>...</svg>  或 null",
      "audio_text": "听力题原文（仅英语听力题）  或 null",
      "group_id": "g6_chinese_r3_reading_01  或 null",
      "group_order": 1
    }
  ]
}
```

**字段说明**：
- `subject`：Dart enum 标识符（小写英文）：`chinese / math / english / physics / chemistry`
- `grade`：6/7/8/9 整数
- `chapter`：必须严格匹配 `lib/database/curriculum_seed.dart` 中该年级该科目的章节名
- `knowledge_point`：`<category>/<name>` 路径形式，必须从 `lib/database/knowledge_points_seed.dart` 选；新批次引入的新 KP 通过 `knowledge_points_added` 字段一并入库
- `type`：仅这三个字符串
- `difficulty`：题目本身的复杂度（**仅数据层使用，UI 不显示**——V3.8.2 起 UI 仅按 round 显示难度档）
- `round`：**必填** 1-4。UI 显示为「基础/中等/较难/竞赛」。
- `options`：仅 type=choice 时填，4 项 `A./B./C./D.` 前缀
- `answer`：choice 题填字母（A/B/C/D）；fill/calculation 填答案文本，公式题用 `|||` 分隔等价写法
- `image`：SVG 字符串（以 `<svg` 开头）或 base64 data URL；几何/电路图用
- `audio_text`：英语听力朗读原文。**听力题的 content 字段不能含听力原文**，只能写引导语（如"听以下对话回答问题"）+ 题目本身
- `group_id`：**V3.8.2 新增**。系列题（阅读理解 cluster / 多考点连贯题）共享同一 group_id，不同 group_order 表示顺序。抽到 group 内任一题，App 会把同 group 全部题一起抽 + 按 group_order 排序展示。**单独题**此字段省略或 null。
- `group_order`：**V3.8.2 新增**。group 内题目顺序，从 1 开始。无 group 时省略

## 5. KP 纪律（最关键）

**强制流程：每生成新批次前**：
1. 读 `lib/database/knowledge_points_seed.dart` 确认目标 (subject, grade) 的 KP 清单存在
2. **不存在则先列**：约 25-50 个二级 KP，按一级类目组织
3. 题目 `knowledge_point` 字段必须从清单中选，不允许自由填
4. 命名风格：`<一级类目>/<二级具体名>`（如 `比和比例/化简比`）
5. 同一概念跨年级用同名 KP（如 `修辞/比喻` 在六七八九通用）

**新 KP 通过 batch JSON 的 `knowledge_points_added` 字段同步入库**：app 端导入时合并到 `knowledge_points` 表。

## 6. 题目质量纪律

### 输入法限制（含这些字符的填空题必须改选择题）

- 数学符号：`π`、`²`/`³` 上下标、`×÷`、复杂分式（`1/3πr²h`）、根号 `√`
- 拼音声调：`á à ǎ ē é ě è` 等
- 希腊字母（除 π）：`α β γ θ φ`
- 化学下标：`H₂O`、`CO₂`
- 任何需切换 / 复制粘贴才能输入的符号

### 填空题只允许的答案类型

- 纯数字：`62.8`、`40`、`3.14`
- 简单分数：`1/3`、`2/3`
- 简单比例：`3:1`、`2:3`
- 单字 / 短词（≤4 字）
- 英文短词（≤8 字母）

**含限制项的题必须重写为 choice 4 选项。**

### 禁开放题

不收"描写__应该写什么"、"举例说明..."这类无唯一答案题。

### 答案唯一可判定（强化）

fill 题的答案必须满足：
- **直接关键词**：答案必须是阅读材料/课文里直接出现的词、定义、固定搭配；不接受概括/总结/复述
- **不是问"主旨"/"中心"**：这类必须改 choice
- **不是模糊描述**：不收 "突出热闹"、"惊慌失措"、"真情实感" 等带括号注释或带主观描述的答案
- **判定即对错明确**：你（出题人）能瞬间认出"这道题就这一种答案"才能用 fill；犹豫"这答案行不行"立刻改 choice

例（违规）：
- ❌ `"answer": "突出热闹（具体描写）"`
- ❌ `"answer": "惊慌失措"`（人物态度的"标准答案"主观）
- ✅ `"answer": "排比"`（修辞类只此一答）
- ✅ `"answer": "李白"`（作家）
- ✅ `"answer": "梅须逊雪三分白"`（古诗背诵原文）

### 课文/名著类语文题：默认 choice

涉及人物态度 / 情节复述 / 作家 / 作品 / 主题理解 等，**统统出 choice**（4 选项混淆）。仅古诗词背诵原文 + 字音字形纠错可以 fill。

### 阅读理解必须有材料

阅读理解题必须包含完整阅读材料（200-500 字短文）+ 至少 3 道连续小题；每个小题在 content 字段重复带材料以独立可读。每个小题答案明确（fill 答关键词，choice 答 A/B/C/D）。

### 选项混淆度（按难度档调整）

- R1：选项区分明显（一对三错，错答都明显错）
- R2：错选项是同领域近义词（如修辞题用其他三种修辞）
- R3：错选项是 R2 + 部分正确但有偏差（如时态题错选项时态对但人称错）
- R4：错选项有相当合理性，需要仔细推敲

### 数学/物理/化学公式 LaTeX

**所有理科题** content + explanation 中公式**必须用 `$...$` 包 LaTeX**：
- 分数：`$\frac{1}{2}$`
- 上标：`$r^2$`
- 希腊字母：`$\pi$` `$\alpha$` `$\beta$`
- 乘除：`$\times$` `$\div$`
- 化学下标：`$H_2O$` `$CO_2$`
- 化学方程式：`$2H_2 + O_2 \rightarrow 2H_2O$`

例：`化简 $\frac{1}{2}:\frac{1}{4}$ = ?`、`求 $H_2SO_4$ 中 S 的化合价`

### 理科图形 SVG（image 字段）

- **数学**：几何图、坐标系、立体图、统计图
- **物理**：受力分析图、电路图、光路图、运动图
- **化学**：实验装置图、分子结构图、反应流程图

每个含图的题在 `image` 字段填 SVG 字符串（以 `<svg` 开头）。SVG 用 `viewBox` + 黑色描边，简洁清晰。

### 公式题 alt_answers

涉及多种等价写法的公式题（如圆锥体积），`answer` 字段用 `|||` 分隔：
```
"answer": "πr²h/3|||pi*r*r*h/3|||pi*r^2*h/3|||(1/3)*π*r²*h"
```

UI 显示用第一段（`displayAnswer = answer.split('|||').first`）。

## 7. 难度档 Rubric

每题必须按 round 分档，标准如下。

### V3.8.2 起的难度跨档单调保证（重要新规）

**规则**：R(n+1) 内最简单题必须 ≥ R(n) 内最难题。即每档 round 是真正"批次升级"，不能跨档倒挂。

具体翻译为新基线：
- **R1 新基线**：等于 V3.6/V3.7 R1 内 hard 题难度（即原 R1 的 80easy/80medium/40hard 中的 hard 档）
- **R2 新基线**：等于 V3.7.6 R2 内 hard 题难度
- **R3 新基线**：明显高于 R2 hard
- **R4 新基线**：明显高于 R3

**对 cron 出题人的指令**：
- 出 R1 题时，思考"如果这道题放在 V3.6 R1 batch 里算 hard 题吗？" 算才出
- 出 R3 题时，思考"如果让做过 V3.7.6 R2 hard 题的孩子做这道，明显比那个难吗？" 明显才出
- difficulty 字段（easy/medium/hard）仅作为 round 内的细分标签使用（数据分布需要），UI 不显示

### 数学

- **R1 基础**：综合应用题（含 2 步推理）/ 多概念基础应用（**= V3.6 R1 hard**）
- **R2 中等**：3-4 步综合 / 单位换算 / 中等应用题（**= V3.7.6 R2 hard**）
- **R3 较难**：多概念综合 + 隐藏条件 / 几何代数交叉 / 复杂应用题
  - **选择题挑战度**：错选项要有"看似合理但忽略某条件"的设计，不是简单数字干扰
- **R4 竞赛**：思维拓展 / 多解 / 反证 / 数论拓展 / 跨章节综合
  - **选择题挑战度**：错选项需要正向算才能排除，不能瞄一眼就排除

### 语文

- **R1**：字音字形辨析 / 综合修辞识别 / 课文细节理解（= V3.6 R1 hard）
- **R2**：短中篇阅读理解 / 综合修辞应用 / 文学常识对比 / 病句修改（= V3.7.6 R2 hard）
- **R3**：长篇阅读理解 + 多小题 / 古诗鉴赏（情感+手法+意境）/ 文言翻译 / 跨课文比较
- **R4**：经典深度赏析 / 文化典故拓展 / 古诗文综合

### 英语（每档基线对应 V3.6/V3.7.6 hard 档）

- **R1**：基础词汇 / 时态简单变形 / 简单句型
- **R2**：句型转换 / 复合时态 / 中等阅读
- **R3**：长篇阅读 / 写作 / 语境推断 / 高频固定搭配
- **R4**：高阶语法 / 深度写作框架 / 同义辨析

### 物理

- **R1**：概念辨析 / 单一公式套用
- **R2**：2 步公式综合 / 实验现象分析
- **R3**：复合运动 / 实验设计 / 多公式综合
- **R4**：综合实验 / 复杂场景 / 公式推导

### 化学

- **R1**：元素符号 / 基础反应识别
- **R2**：基本方程式书写 / 简单计算
- **R3**：综合方程 / 多步推断
- **R4**：实验设计 / 综合工业应用 / 综合推断题

## 8. 题型 / 难度 分布建议（每科 200 题）

- **题型**：80 选 + 80 填 + 40 算（数学/物理/化学）；120 选 + 80 填（语文/英语，无 calc）
- **difficulty 字段（题目内在复杂度）按 round 限制**：
  - **R1（基础）**：80 easy + 80 medium + 40 hard
  - **R2（中等）**：40 easy + 100 medium + 60 hard
  - **R3（较难）**：**0 easy** + 120 medium + 80 hard ← R3 不出 easy 题
  - **R4（竞赛）**：**0 easy** + 60 medium + 140 hard ← R4 大头是 hard
- **理由**：用户全局选 R3（"较难"）时不希望抽到题目内在简单的题。round 是"批次档" + difficulty 是"题目内在难度"，二者必须配合，不能正交导致简单题混入高 round 批次。

## 9. 工作流（每次 cron 启动后必做）

1. **解析当前日期**：用 Bash `date -u +%Y-%m-%d` 拿 UTC 日期；北京时间 = UTC+8
2. **查阶段日程**：本文件第 2 节，找今天对应任务（年级 / 科目 / round）
3. **读 KP 清单**：`lib/database/knowledge_points_seed.dart`，确认目标 KP 存在；不存在先列
4. **并行生成各科**：用 Agent 并行（每科一个 Agent），每 Agent 生成 200 题
   - 严格遵守纪律：输入法限制 / LaTeX / KP 严格匹配 / round 标记
5. **写文件**：每个 batch 双写
   - `assets/data/batches/<filename>.json`
   - `question_bank/<filename>.json`
6. **注册新 batch**：
   - 更新 `question_bank/index.json` 加入条目
   - 更新 `lib/main.dart` 的 `_bundledBatchAssets` 加入路径
7. **flutter analyze**（确保无 error）
8. **commit + push**：commit message 格式 `Cron YYYY-MM-DD: <年级 R<round>> <科目列表> 各 200 题`
9. **回报**：在 issue tracker / 提交一个简短日志（commit hash + 总题数）

## 9.5 V3.8.2 起的"已掌握题"纪律

每题用户累计答对 ≥3 次后被 app 自动从抽题池排除（标记为"已掌握"）。

学情 JSON 的 `by_kp` 段含 `mastered_count`（该 KP 下被排除的题数）。

**Cron 出题指引**：
- KP 的 mastered_count 高 + 该 KP 总题数少 → 该 KP 即将"无题可抽"，**优先生成同 KP 新变体题**避免出现"该知识点的题练完了"
- 此规则覆盖第二/三阶段所有出题决策（不只是错题反馈池）

## 10. 第二/三阶段错题反馈（5/20 起）

阶段 2/3 在第 4 步前**额外**做：

1. clone 私有 repo `daughter-learning-data`
2. 找 `learning_data/{any-deviceName}/*.json` 最近 7 天文件
3. 聚合所有用户的 `by_kp` 段：按 `(subject, knowledge_point)` 求和 attempts/errors，重算 error_rate
4. 仅纳入 `attempts ≥ 3` 的 KP
5. 30% 反馈池题数按 error_rate 加权分配到这些 KP（生成相似变体题，按错题原 round 出）
6. 70% 池继续按章节循环铺新内容

## 11. 错误处理

- 单科生成失败 → 跳过该科，继续其他科目；commit 时只 push 成功的
- KP 清单缺失 → 立即先生成清单 + commit + 接着生成题包
- flutter analyze 报 error → 不 push，写 issue 报告
- repo 拉取失败 → 重试 3 次，仍失败则报警

**Cron 端处理规则：**

1. **同模式 ≥3 次 approved（appeal 类）**：
   - 提示 "AnswerMatcher 归一化规则待调"。例：所有"x=1, y=2, z=3"格式的多变量答案被申诉通过 → 提示加多变量等价归一化。
   - 提交一个 `cron-issue.md` 给 Famin，**不要**自行修改 `lib/utils/answer_matcher.dart`。

2. **单题 approved**：
   - 该题包标记 `_review_flagged`，下次再生成同 KP 题包时优先回避同模式表达。
   - 如评分类型是 `subjective_grading` 且 score ∈ {pass, good, perfect} → 该题模式可继续生成，但 `parent_note` 内容要喂回新一批 prompt 作为质量校准信号。

3. **rejected**：不动作（说明判定本身正确，题包没问题）。

4. **历史 review_feedback 累计**：cron 端建一个 `learning_data/review_history.json` 滚动累计，避免单日 JSON 丢失记忆。

5. **隐私**：`child_note` / `parent_note` 不上 GitHub 任何公开 repo（学情 repo 是私有，OK）。生成新批次的 commit message 不暴露申诉具体内容。
