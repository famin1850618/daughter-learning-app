# 语文难度 rubric（活文档，V2 起）

> **作用：** 入库 reviewer / agent 判 round 时**必读**，与 `docs/anchor_questions_g6_chinese.json` 配合使用。
> **维护：** Famin verify 反馈累积，每轮新增/调整后增量更新。
> **不能脱离 anchors 单独用。** Reviewer 每题判定时必须引用 anchor_id 或本文件具体条目。
> **当前阶段：** 0→1 升级中（首轮 verify 完成 + 锚点修复完成 = 阶段 1 触发）。

---

## 1. 4 档总览

| 档 | 名 | 触发特征 |
|---|---|---|
| R1 | 基础 | 直接背诵默写 / 课内字音字形 / 国别作家匹配 / 课内必修常识 / 候选池窄（2-3 选项明确） |
| R2 | 中等 | 近义词辨析（4 候选搭配差异）/ 病句识别（关联词搭配错）/ 标点综合甄别 / 句式转换规则套用 / 修辞基础三种识别 |
| R3 | 较难 | 全文主旨概括 / 文言人物态度推断 / 跨文化常识跨文体辨析 / 长说明文指代精读 / 段意综合（含直接引语） |
| R4 | 竞赛 | 选项嵌套式包含关系（A⊂B⊂C 偏序集合）/ 多空虚词修辞鉴赏（互不可换）/ 4 阶段情感曲线 / 4 古诗对仗多维核对 / 散文段落承上启下 |

## 2. Famin 关键判定信号（V1 verify 提炼，强制应用）

### 2.1 降档信号（agent 判 R(n+1)，Famin 实判 Rn）

| 信号 | 反馈来源 | 应用 |
|---|---|---|
| **褒贬明显** | r3_a1 R3→R1（"褒贬明显，选项无任何迷惑性"）| 选项中正/反义对比明显（如"实事求是 vs 不懂装懂"），错答可瞬间排除 → 降到 R1 |
| **段意直白** | r3_a2 R3→R2（"理解直白，选项迷惑性不高"）| 阅读理解文段背景+态度都已明示，无需透过事件层提炼 → R2 不到 R3 |
| **三短一长选最长** | r4_a2 R4→R2（"三短一长选最长，选项无迷惑性"）| 4 选项中 1 个明显比其他长，无嵌套关系 → 降到 R2，不当 R3/R4 |
| **候选池窄** | （V3.12.5 reviewer 反推）| 给定 5 个固定四字成语候选（如 #8-12 同型），即使释义反推也只 R1 |
| **课内必修常识** | r1_a4（笛福、马克吐温等）| 不论作家姓名长短，课内必修一律 R1 |

### 2.2 升档信号（agent 判 R(n)，Famin 实判 R(n+1) 或更高）

| 信号 | 反馈来源 | 应用 |
|---|---|---|
| **选项嵌套式集合关系** | r4_a2 反推（r4_a2 因无嵌套被降档，反向推论：嵌套 = R4）| 当 A⊂B 或 D=B+C 等嵌套关系出现，必须正向算才能区分 → R4 |
| **多空虚词鉴赏** | r4_a1（"莫不"三空互不可换）| 同一虚词三次复现 + 多空答案不可互换 → R4（鉴赏评价层）|
| **跨文化常识 4 选 1** | r3_a5（学弈/骑鹅/汤姆/荀子）| 4 项均含课内+课外细节需逐项核对 → R3 |
| **古诗对仗鉴赏** | r4_a4（4 古诗工对甄别）| 跨 4 首古诗逐句拆字数+词性+结构+平仄 → R4 |
| **散文段落衔接** | r4_a5（《买馒头》⑫⑬）| 整篇散文 14 段 + 抒情句嵌入承上启下位置 → R4 |
| **4 阶段情感曲线** | r4_a3（《向日葵》4 阶段，4 选项每个错 1 空）| 多阶段情感对位 + 选项每个部分对 → R4 |

### 2.3 拿不准信号（Famin 标 unsure，处置方式）

| 信号 | 处置 |
|---|---|
| **题面不附原文 / 阅读理解原文未呈现** | reviewer 谨慎降档评估，不要默认升 R3/R4。Famin r3_a4 / r4_a3 / r4_a5 都是因为没看到原文判定保守 |
| **agent 评 R(n) Famin comment 提"或许 R(n+1)"或"最多 R(n-1)"** | 标 unsure，不当强锚点用，但保留作软参考 |

## 4. Reviewer 调用纪律（阶段 1）

每题判定 reasoning 必须包含：
1. 引用最相近的 `anchor_id`（在 round±1 内）
2. 引用本文件 §2.1 / §2.2 哪条降档/升档信号（如适用）
3. **不许凭空 LLM 判**

输出 jsonl 字段：
- `current_round` / `suggested_round` / `verdict` / `reasoning` / `anchor_used` / `rubric_signals_applied[]`
